# 記憶體與快取感知策略

## 快速參考：重要事實

- **記憶體階層** 從 CPU 暫存器（最快）延伸到片外記憶體（最慢）
- **Cache line** 通常為 32、64 或 128 位元組 — 存取一個位元組就會載入整條 cache line
- **時間局部性** 指重複使用最近存取的資料；**空間局部性** 指存取鄰近的資料
- **記憶體池** 消除碎片化並提供可預測的配置效能
- **偽共享（False sharing）** 發生在不同執行緒存取位於同一條 cache line 上的變數時
- **結構體封裝** 透過排列欄位以減少對齊間隙來最小化填充
- **AoS vs. SoA** — 結構體陣列 vs. 陣列結構體，適用於不同的存取模式
- **記憶體對齊** 確保最佳存取效能（通常為 4、8 或 16 位元組）

## 嵌入式系統中的關鍵資源

嵌入式系統中的記憶體最佳化特別重要，因為記憶體通常是最受限的資源。與可以透過時脈頻率或平行處理來擴展的運算資源不同，記憶體容量是固定的，無法輕易增加。這使得高效的記憶體使用對於建構能在可用限制條件內處理所需工作負載的系統至關重要。

記憶體使用的最佳化始於理解記憶體階層以及不同類型的記憶體如何與系統互動。片上記憶體，例如 cache 和暫存記憶體（scratchpad memory），提供最快的存取速度但容量有限。片外記憶體，例如 DRAM 和 flash，提供更大的容量但代價是更高的延遲和功耗。有效使用此階層需要仔細放置資料和程式碼，以最大化快速記憶體的使用，同時最小化對較慢記憶體的存取需求。

## 核心概念

### **概念：記憶體階層影響**
**重要性**：理解每個記憶體層級的速度和容量權衡，對於最佳資料放置和存取模式至關重要。

**最簡範例**：
```c
// 資料放置中的記憶體階層意識
typedef struct {
    uint32_t critical_data[16];    // 放在 L1 cache
    uint32_t frequent_data[64];    // 放在 L2 cache
    uint32_t occasional_data[256]; // 放在主記憶體
} data_layout_t;

// 存取模式最佳化
void process_data(data_layout_t *data) {
    // 先處理關鍵資料（留在 L1）
    for (int i = 0; i < 16; i++) {
        data->critical_data[i] *= 2;
    }
    
    // 再處理頻繁資料（對 L2 cache 友善）
    for (int i = 0; i < 64; i++) {
        data->frequent_data[i] += 1;
    }
}
```

**動手試試**：對不同記憶體層級的記憶體存取時間進行效能分析。

**重點整理**：將頻繁存取的資料放在較快的記憶體層級。

### **概念：Cache Line 行為**
**重要性**：Cache line 是 cache 運作的基本單位 — 理解其大小和行為可實現最佳記憶體存取模式。

**最簡範例**：
```c
// Cache line 最佳化
#define CACHE_LINE_SIZE 64

// 將資料對齊到 cache line 邊界
typedef struct {
    uint32_t data1 __attribute__((aligned(CACHE_LINE_SIZE)));
    uint32_t data2 __attribute__((aligned(CACHE_LINE_SIZE)));
} cache_aligned_t;

// 在多執行緒程式碼中避免偽共享
typedef struct {
    uint32_t thread1_data __attribute__((aligned(CACHE_LINE_SIZE)));
    uint32_t thread2_data __attribute__((aligned(CACHE_LINE_SIZE)));
} thread_safe_t;
```

**動手試試**：測量 cache line 對齊與未對齊的效能影響。

**重點整理**：將頻繁存取的資料對齊到 cache line 邊界，以最小化 cache miss。

### **概念：記憶體存取模式**
**重要性**：循序存取模式透過空間局部性最大化 cache 效能，而隨機存取則會導致過多的 cache miss。

**最簡範例**：
```c
// 好的做法：循序存取（對 cache 友善）
void process_array_good(uint32_t arr[], int size) {
    for (int i = 0; i < size; i++) {
        arr[i] = process_element(arr[i]);  // 循序存取
    }
}

// 不好的做法：隨機存取（對 cache 不友善）
void process_array_bad(uint32_t arr[], int indices[], int size) {
    for (int i = 0; i < size; i++) {
        int idx = indices[i];  // 隨機索引
        arr[idx] = process_element(arr[idx]);  // 隨機存取
    }
}
```

**動手試試**：比較循序與隨機陣列存取模式的效能。

**重點整理**：盡可能設計資料結構和演算法以實現循序記憶體存取。

## 記憶體階層理解

現代嵌入式系統具有複雜的記憶體階層，橫跨多個層級，每個層級在速度、容量和功耗方面具有不同的特性。理解此階層對於有效的記憶體最佳化至關重要。該階層通常包括：

- **CPU 暫存器**：最快存取、最小容量、最高功耗
- **L1 Cache**：極快存取、小容量、高功耗
- **L2 Cache**：快速存取、中等容量、中等功耗
- **L3 Cache**：中等存取速度、較大容量、較低功耗
- **片上記憶體**：中等存取速度、中等容量、低功耗
- **片外記憶體**：慢速存取、大容量、最低功耗

每個層級的效能特性差異顯著。CPU 暫存器提供單週期存取，而片外記憶體可能需要數百個週期。功耗差異也很大，暫存器每次存取消耗最多功率，片外記憶體消耗最少。這產生了一個複雜的最佳化問題，必須根據存取頻率和效能需求將資料放置在適當的層級。

## 快取感知程式設計：最大化記憶體效能

快取感知程式設計代表最精密的最佳化技術之一，需要深入理解現代記憶體系統的運作方式。現代處理器中的 cache 階層旨在彌合處理器速度和記憶體速度之間的差距，但其效能完全取決於軟體存取記憶體的方式。不良的記憶體存取模式可能使即使是最精密的 cache 系統也變得無效。

快取感知程式設計的基本原則是參考局部性，分為兩種形式：時間局部性和空間局部性。時間局部性是指程式傾向於在一段時間內重複存取相同的記憶體位置。這通過將頻繁存取的資料保留在 cache 中來利用。空間局部性是指程式傾向於存取彼此相近的記憶體位置。這通過在存取鄰近位置時預取資料到 cache 中來利用。

快取感知資料結構使用多種技術來最大化 cache 效能。資料結構配置可以透過將相關資料分組在一起並排列資料以最小化 cache line 碎片化來最佳化空間局部性。存取模式可以透過盡可能重複使用資料並避免不必要的記憶體存取來最佳化時間局部性。記憶體配置可以最佳化以利用 cache line 大小和記憶體對齊需求。

## 記憶體配置策略

記憶體配置策略在記憶體最佳化中扮演關鍵角色。動態記憶體配置雖然方便，但可能導致碎片化和不可預測的效能。靜態配置提供可預測的效能，但當不同元件有不同的記憶體需求時可能會浪費記憶體。記憶體池配置提供了一個折衷方案，既具有動態配置的靈活性，又具有靜態配置的可預測性。

記憶體池配置對嵌入式系統特別有效，因為它消除了碎片化並提供可預測的記憶體使用模式。記憶體池在系統啟動時預先配置，並劃分為固定大小的區塊。當需要記憶體時，從適當的記憶體池中配置一個區塊。當記憶體被釋放時，它會返回記憶體池以供重複使用。這種方法確保不會發生記憶體碎片化，並提供可預測的效能特性。

堆疊配置是另一種針對暫時記憶體需求的有效策略。堆疊配置非常快速，並在變數超出作用域時自動處理釋放。然而，嵌入式系統中的堆疊大小通常有限，因此堆疊配置應謹慎用於小型、暫時的資料結構。

## 資料結構記憶體配置

資料結構的記憶體配置對 cache 效能和整體記憶體效率有顯著影響。不良的資料結構配置可能導致過多的 cache miss，並因填充和對齊需求而浪費記憶體。最佳化資料結構配置可以在最小程式碼變更的情況下提供顯著的效能提升。

結構體封裝是一種透過排列結構體欄位以最小化填充來減少記憶體浪費的技術。在 C/C++ 中，編譯器可能在結構體欄位之間插入填充以確保目標架構的正確對齊。透過仔細排序結構體欄位，通常可以最小化或完全消除填充。這對於頻繁配置或儲存在大型陣列中的結構體尤其重要。

結構體陣列（AoS）與陣列結構體（SoA）是資料結構設計中另一個重要考量。AoS 配置將一個結構體的所有欄位儲存在一起，這對於存取單一結構體的所有欄位很好，但對於跨多個結構體存取單一欄位則較差。SoA 配置將單一欄位的所有值儲存在一起，這對於跨多個結構體存取單一欄位很好，但對於存取單一結構體的所有欄位則較差。

## 記憶體存取模式最佳化

記憶體存取模式的最佳化與記憶體使用的最佳化同樣重要。現代處理器嚴重依賴 cache 來維持效能，低效的記憶體存取模式可能大幅降低 cache 效能。循序存取模式通常比隨機存取模式表現好得多，資料結構的對齊也會對記憶體效能產生顯著影響。

循序記憶體存取是 cache 效能的最佳選擇，因為它最大化了空間局部性。當資料被循序存取時，處理器可以在需要之前將資料預取到 cache 中，減少 cache miss 並提升效能。另一方面，隨機存取模式提供較差的空間局部性，並可能導致過多的 cache miss。

記憶體對齊是記憶體存取效能的另一個關鍵因素。現代處理器以對齊的區塊存取記憶體，通常一次 4、8 或 16 位元組。未對齊的存取可能導致處理器執行多次記憶體存取並合併結果，顯著降低效能。透過仔細的資料結構設計和記憶體配置策略通常可以確保正確的對齊。

## Cache Line 最佳化

Cache line 是 cache 運作的基本單位，針對 cache line 行為進行最佳化可以提供顯著的效能提升。Cache line 通常為 32、64 或 128 位元組，當存取單一位元組時，整條 cache line 都會被載入 cache。理解這種行為可以讓程式最佳化記憶體存取模式。

偽共享（False sharing）是多執行緒應用程式中的常見問題，不同的執行緒存取恰好位於同一條 cache line 上的不同變數。當一個執行緒修改一個變數時，整條 cache line 被無效化，導致其他執行緒必須從記憶體重新載入資料，即使它們存取的是不同的變數。這可以透過確保頻繁存取的變數不在同一條 cache line 上來避免。

Cache line 對齊也可以透過確保資料結構從 cache line 邊界開始來提升效能。這可以減少存取資料結構時需要載入的 cache line 數量，特別是對於頻繁存取的資料。然而，過度的對齊可能浪費記憶體，因此必須仔細考量其權衡。

## 記憶體池管理

記憶體池管理是一種精密的記憶體最佳化技術，對於具有可預測記憶體使用模式的系統可以提供顯著的效能提升。記憶體池是預先配置的記憶體區域，被劃分為固定大小的區塊。這種方法消除了碎片化，並提供可預測的配置和釋放效能。

記憶體池的設計涉及多項考量。區塊大小應根據典型配置物件的大小來選擇，如有必要可使用多個記憶體池對應不同的大小範圍。記憶體池大小應根據同時配置的最大物件數量來選擇。記憶體池可以使用簡單的鏈結串列或更精密的資料結構來實作，取決於效能需求。

記憶體池的配置和釋放通常是非常快速的操作，往往只需要幾條指令。這使得記憶體池非常適合需要可預測效能的即時系統。然而，記憶體池不支援可變大小的配置，因此在物件大小可預測或可以歸類為少數大小等級時最為有效。

## 垃圾回收與記憶體管理

在使用動態記憶體配置的系統中，垃圾回收和記憶體管理策略可能對效能產生顯著影響。傳統的垃圾回收演算法可能導致不可預測的暫停和過多的記憶體使用，這在嵌入式系統中通常是不可接受的。替代方法如參考計數、標記-清除和分代垃圾回收可以提供更好的效能特性。

參考計數是一種簡單的垃圾回收技術，維護每個物件的參考計數。當參考計數歸零時，物件立即被釋放。這種方法提供可預測的效能，但對於頻繁被參考的物件可能產生過多的開銷，且無法處理循環參考。

標記-清除垃圾回收是一種更精密的方法，可以處理循環參考但可能導致不可預測的暫停。這種方法通過標記所有可達的物件，然後掃過記憶體以釋放未標記的物件來運作。標記階段可以最佳化以最小化暫停時間，但通常無法完全避免某些暫停。

## 記憶體映射 I/O 最佳化

記憶體映射 I/O 是嵌入式系統中的常見技術，透過記憶體位址存取硬體暫存器。最佳化記憶體映射 I/O 存取可以顯著提升系統效能，特別是對於頻繁存取的暫存器。

volatile 修飾詞對於記憶體映射 I/O 至關重要，可以防止編譯器最佳化掉暫存器存取。然而，過度使用 volatile 可能阻礙其他最佳化，因此 volatile 應僅在必要時使用。暫存器存取模式也應進行最佳化，以最小化記憶體存取次數並利用處理器特定的功能。

暫存器快取可以作為頻繁存取暫存器的有效最佳化。不必每次需要暫存器值時都存取硬體暫存器，而是可以將值快取在區域變數中，僅在必要時更新。這種方法可以顯著減少記憶體存取開銷，但必須注意確保快取值與實際暫存器值保持一致。

## 引導式實驗

### 實驗 1：記憶體階層效能分析
1. **設置**：建立不同大小的資料結構
2. **分析**：測量不同記憶體層級的存取時間
3. **分析**：識別資料何時在 cache 層級之間移動
4. **最佳化**：將關鍵資料放在較快的記憶體中

### 實驗 2：Cache Line 對齊影響
1. **實作**：對齊與未對齊的資料結構
2. **測量**：使用效能計數器測量 cache miss 率
3. **分析**：不同對齊策略的執行時間
4. **結論**：對齊在何時最為重要？

### 實驗 3：記憶體存取模式分析
1. **建立**：循序 vs. 隨機存取模式
2. **分析**：使用 cache 分析工具（perf、valgrind）
3. **測量**：不同模式的效能影響
4. **最佳化**：重新建構演算法以獲得更好的局部性

## 結論

記憶體與快取感知策略為嵌入式系統中高效記憶體使用奠定了基礎。理解記憶體階層並針對 cache 行為進行最佳化，可以提供比演算法最佳化更為顯著的效能提升。關鍵在於設計與記憶體系統協同運作而非對抗的資料結構和存取模式。

最有效的記憶體最佳化方法結合了多種技術：仔細的資料結構設計、最佳化的記憶體配置，以及快取感知的存取模式。每種技術都能提供漸進式的改善，但組合使用可以提供顯著的整體效能增益。具體使用哪些技術取決於目標系統架構和應用程式的特定效能需求。

隨著嵌入式系統變得更加複雜和記憶體受限，有效記憶體最佳化的重要性只會持續增加。記憶體技術和最佳化技術的持續發展將提供改善記憶體效能的新機會，但理解記憶體階層和針對 cache 行為進行最佳化的基本原則將始終是有效記憶體最佳化的基石。

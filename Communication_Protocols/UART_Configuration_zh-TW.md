## 🎯 **概述**

UART 組態與設定是嵌入式系統中可靠串列通訊的基礎。正確的組態可確保最佳效能、無錯誤通訊，以及在各種硬體平台和通訊需求下的高效資源利用。

### **核心概念**
- **硬體初始化** - GPIO 組態、時脈設定、周邊組態
- **緩衝策略** - 環形緩衝區、中斷驅動緩衝、DMA 緩衝
- **中斷處理** - ISR 設計、中斷優先權、巢狀中斷
- **錯誤管理** - 錯誤偵測、復原機制、逾時處理
- **效能最佳化** - 鮑率選擇、緩衝區大小調整、中斷最佳化

## 🧠 **概念優先**

### **緩衝策略**
**概念**：不同的緩衝方式在嵌入式系統中有不同的取捨。
**重要性**：正確的緩衝策略可以決定通訊是可靠的還是會發生資料遺失。
**最小範例**：比較 UART 的中斷驅動、輪詢與 DMA 緩衝方式。
**動手做**：實作不同的緩衝策略，並測量效能與可靠性。
**重點歸納**：中斷驅動最為常見，DMA 最適合高速傳輸，輪詢最簡單但效率最低。

### **中斷與 DMA**
**概念**：中斷提供靈活性，DMA 提供大量傳輸的效率。
**重要性**：選擇正確的方式會影響效能與系統回應能力。
**最小範例**：分別使用中斷和 DMA 方式實作 UART 接收。
**動手做**：測量不同傳輸大小下的 CPU 使用率與延遲。
**重點歸納**：大量傳輸使用 DMA，小型/頻繁傳輸使用中斷，或結合兩者以達最佳效能。

## 🤔 **什麼是 UART 組態？**

UART 組態涉及設定通用非同步收發傳輸器的硬體和軟體元件，以建立可靠的串列通訊通道。它涵蓋硬體初始化、參數組態、緩衝策略及錯誤處理機制。

### **核心概念**

**硬體初始化：**
- **GPIO 組態**：設定發送和接收腳位
- **時脈管理**：配置周邊時脈和鮑率產生
- **周邊設定**：初始化 UART 暫存器和控制結構
- **中斷組態**：設定中斷來源和優先權

**通訊參數：**
- **鮑率**：以每秒位元數計算的資料傳輸速度
- **資料格式**：資料位元數、同位元檢查和停止位元
- **流量控制**：硬體或軟體流量控制機制
- **時序**：時脈同步和取樣策略

**緩衝與管理：**
- **資料緩衝**：接收和發送資料的暫存空間
- **中斷處理**：高效處理通訊事件
- **錯誤偵測**：識別和處理通訊錯誤
- **流量控制**：管理資料流以防止緩衝區溢位

### **UART 通訊流程**

**發送過程：**
```
應用程式資料 → 緩衝區 → UART 硬體 → 實體線路 → 接收端
     ↑              ↑           ↑              ↑
   軟體層        記憶體層      硬體層        電氣層
```

**接收過程：**
```
實體線路 → UART 硬體 → 緩衝區 → 應用程式資料
     ↑              ↑           ↑           ↑
   電氣層        硬體層      記憶體層      軟體層
```

**組態層級架構：**
```
系統層級
    ├── 時脈組態
    ├── GPIO 設定
    └── 周邊啟用
    
UART 層級
    ├── 鮑率
    ├── 資料格式
    ├── 流量控制
    └── 中斷設定
    
應用層級
    ├── 緩衝區管理
    ├── 錯誤處理
    └── 協定實作
```

## 🧠 **UART 組態概念**

### **硬體架構**

**UART 方塊圖：**
```
┌─────────────────────────────────────────────────────────────┐
│                    UART 周邊                                │
├─────────────────┬─────────────────┬─────────────────────────┤
│   發送器        │    接收器       │      控制邏輯           │
│                 │                 │                         │
│  ┌───────────┐  │  ┌───────────┐  │  ┌─────────────────────┐ │
│  │ TX 緩衝區 │  │  │ RX 緩衝區 │  │  │   組態暫存器        │ │
│  │           │  │  │           │  │  │                     │ │
│  └───────────┘  │  └───────────┘  │  └─────────────────────┘ │
│        │        │        │        │           │              │
│  ┌───────────┐  │  ┌───────────┐  │  ┌─────────────────────┐ │
│  │ TX 移位   │  │  │ RX 移位   │  │  │   狀態與控制        │ │
│  │ 暫存器    │  │  │ 暫存器    │  │  │                     │ │
│  └───────────┘  │  └───────────┘  │  └─────────────────────┘ │
│        │        │        │        │           │              │
│  ┌───────────┐  │  ┌───────────┐  │  ┌─────────────────────┐ │
│  │ TX 腳位   │  │  │ RX 腳位   │  │  │   中斷控制器        │ │
│  │ 驅動器    │  │  │ 接收器    │  │  │                     │ │
│  └───────────┘  │  └───────────┘  │  └─────────────────────┘ │
└─────────────────┴─────────────────┴─────────────────────────┘
```

**時脈產生：**
- **系統時脈**：UART 周邊的主要時脈來源
- **鮑率產生器**：將系統時脈分頻以產生鮑率
- **取樣時脈**：用於資料取樣和時序的內部時脈
- **過取樣**：每位元多次取樣以提高雜訊免疫力

**資料流：**
- **發送路徑**：應用程式 → TX 緩衝區 → TX 移位暫存器 → TX 腳位
- **接收路徑**：RX 腳位 → RX 移位暫存器 → RX 緩衝區 → 應用程式
- **控制路徑**：組態暫存器 → 控制邏輯 → 狀態暫存器
- **中斷路徑**：狀態暫存器 → 中斷控制器 → CPU

### **組態參數**

**鮑率組態：**
- **鮑率計算**：BR = System_Clock / (16 × UARTDIV)
- **過取樣**：8 倍或 16 倍過取樣以提高雜訊免疫力
- **容差**：最大可接受鮑率誤差（±2-3%）
- **常用速率**：9600、19200、38400、57600、115200、230400、460800、921600

**資料格式組態：**
- **資料位元**：每個字元 7、8 或 9 位元
- **同位元檢查**：無、偶同位元或奇同位元
- **停止位元**：1 或 2 個停止位元
- **字元格式**：起始位元 + 資料位元 + 同位元 + 停止位元

**流量控制組態：**
- **硬體流量控制**：RTS/CTS 或 DTR/DSR 訊號
- **軟體流量控制**：XON/XOFF 字元
- **無流量控制**：簡單的點對點通訊
- **流量控制邏輯**：防止緩衝區溢位和不足

### **緩衝策略**

**緩衝區類型：**
- **線性緩衝區**：用於資料儲存的簡單陣列
- **環形緩衝區**：循環緩衝區，可高效利用記憶體
- **DMA 緩衝區**：直接記憶體存取緩衝區，用於高吞吐量
- **分散-聚集緩衝區**：用於複雜資料的多緩衝區段

**緩衝區管理：**
- **寫入操作**：將資料加入發送緩衝區
- **讀取操作**：從接收緩衝區取出資料
- **溢位保護**：防止緩衝區溢位狀況
- **不足處理**：管理緩衝區不足情況

**中斷驅動緩衝：**
- **TX 中斷**：當 TX 緩衝區為空時觸發
- **RX 中斷**：當 RX 緩衝區有資料時觸發
- **錯誤中斷**：在通訊錯誤時觸發
- **中斷優先權**：管理多個中斷來源

## 🔧 **硬體設定**

### **GPIO 組態**

**腳位指派：**
- **TX 腳位**：發送資料輸出腳位
- **RX 腳位**：接收資料輸入腳位
- **RTS 腳位**：請求發送（流量控制）
- **CTS 腳位**：清除發送（流量控制）
- **DTR 腳位**：資料終端就緒（流量控制）
- **DSR 腳位**：資料集就緒（流量控制）

**GPIO 組態參數：**
- **模式**：UART 腳位的替代功能模式
- **上拉/下拉**：RX 使用內部上拉，TX 不使用
- **驅動強度**：TX 腳位使用高驅動強度
- **速度**：快速鮑率使用高速度
- **替代功能**：UART 專用的替代功能選擇

**訊號調節：**
- **電壓位準轉換**：在不同電壓位準之間轉換
- **雜訊濾波**：濾除雜訊和干擾
- **訊號放大**：放大微弱訊號
- **線路驅動器**：驅動長距離通訊線路

### **時脈組態**

**系統時脈設定：**
- **周邊時脈**：啟用 UART 周邊時脈
- **鮑率時脈**：配置鮑率產生器
- **中斷時脈**：啟用中斷控制器時脈
- **DMA 時脈**：啟用 DMA 控制器時脈（如有使用）

**時脈頻率考量：**
- **系統時脈**：用於鮑率計算的主要頻率
- **鮑率精確度**：確保精確的鮑率產生
- **時脈穩定性**：使用穩定的時脈來源
- **電源管理**：最佳化時脈使用以提高電源效率

**時脈分配：**
- **時脈樹**：了解系統時脈分配
- **時脈閘控**：啟用/停用時脈以進行電源管理
- **時脈多工**：在不同時脈來源之間選擇
- **時脈同步**：同步多個時脈域

## 🔄 **中斷處理**

### **中斷架構**

**中斷來源：**
- **TX 中斷**：發送緩衝區為空中斷
- **RX 中斷**：接收緩衝區非空中斷
- **錯誤中斷**：通訊錯誤中斷
- **狀態中斷**：狀態變更中斷

**中斷處理流程：**
- **中斷向量**：中斷向量表和處理常式
- **中斷上下文**：中斷上下文和堆疊管理
- **中斷返回**：正確的中斷返回和上下文恢復
- **中斷除錯**：除錯中斷相關問題

**中斷優先權：**
- **優先權指派**：為不同中斷指派優先權
- **優先權搶佔**：中斷搶佔和巢套
- **優先權反轉**：避免優先權反轉問題
- **優先權管理**：動態管理中斷優先權

- **錯誤處理**：ISR 中的適當錯誤處理

**ISR 實作：**
- **上下文儲存**：儲存和恢復 CPU 上下文
- **臨界區段保護**：保護 ISR 中的臨界區段

- **ISR 返回**：正確的 ISR 返回和上下文恢復

**ISR 最佳實務：**

```c
// UART 組態參數
typedef struct {
    uint32_t baud_rate;           // 鮑率（每秒位元數）
    uint32_t data_bits;           // 資料位元（7、8、9）
    uint32_t stop_bits;           // 停止位元（1、2）
    uint32_t parity;              // 同位元檢查（NONE、EVEN、ODD）
    uint32_t flow_control;        // 流量控制（NONE、RTS_CTS）
    uint32_t mode;                // 模式（RX_ONLY、TX_ONLY、TX_RX）
    uint32_t oversampling;        // 過取樣（8、16）
} UART_Config_t;

// 使用組態初始化 UART
HAL_StatusTypeDef uart_init(UART_HandleTypeDef* huart, UART_Config_t* config) {
    huart->Instance = USART1;
    huart->Init.BaudRate = config->baud_rate;
    huart->Init.WordLength = config->data_bits == 9 ? UART_WORDLENGTH_9B : UART_WORDLENGTH_8B;
    huart->Init.StopBits = config->stop_bits == 2 ? UART_STOPBITS_2 : UART_STOPBITS_1;
    huart->Init.Parity = config->parity;
    huart->Init.Mode = config->mode;
    huart->Init.HwFlowCtl = config->flow_control;
    huart->Init.OverSampling = config->oversampling == 8 ? UART_OVERSAMPLING_8 : UART_OVERSAMPLING_16;
    
    return HAL_UART_Init(huart);
}
```

### **環形緩衝區實作**

**環形緩衝區結構：**
```c
// 環形緩衝區結構
typedef struct {
    uint8_t* buffer;
    uint16_t size;
    uint16_t head;
    uint16_t tail;
    uint16_t count;
} RingBuffer_t;

// 初始化環形緩衝區
void ring_buffer_init(RingBuffer_t* rb, uint8_t* buffer, uint16_t size) {
    rb->buffer = buffer;
    rb->size = size;
    rb->head = 0;
    rb->tail = 0;
    rb->count = 0;
}

// 寫入環形緩衝區
bool ring_buffer_write(RingBuffer_t* rb, uint8_t data) {
    if (rb->count >= rb->size) {
        return false;  // 緩衝區已滿
    }
    
    rb->buffer[rb->head] = data;
    rb->head = (rb->head + 1) % rb->size;
    rb->count++;
    return true;
}

// 從環形緩衝區讀取
bool ring_buffer_read(RingBuffer_t* rb, uint8_t* data) {
    if (rb->count == 0) {
        return false;  // 緩衝區為空
    }
    
    *data = rb->buffer[rb->tail];
    rb->tail = (rb->tail + 1) % rb->size;
    rb->count--;
    return true;
}
```

   - 鮑率 = System_Clock / (16 × UARTDIV)，用於 16 倍過取樣

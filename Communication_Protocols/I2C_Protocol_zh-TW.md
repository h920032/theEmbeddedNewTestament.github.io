## 🎯 **概述**

I2C（Inter-Integrated Circuit）是一種同步、多主控、多從屬、封包交換、單端、串列通訊匯流排，由 Philips Semiconductor（現為 NXP Semiconductors）發明。它廣泛應用於嵌入式系統中，用於積體電路、感測器及其他周邊裝置之間的通訊。

### **關鍵概念**
- **雙線通訊** - SDA（資料）與 SCL（時脈）線路
- **多主控支援** - 多個主控裝置可控制匯流排
- **定址系統** - 7 位元或 10 位元裝置定址
- **時脈延展** - 從屬裝置可減慢通訊速度
- **仲裁** - 非破壞性仲裁以取得匯流排存取權

### **面試官意圖（他們在探測什麼）**
- 你能解釋開汲極加上拉電阻與上升時間限制嗎？
- 你了解仲裁與時脈延展的行為嗎？
- 你能推論匯流排速度與電容之間的關係嗎？

## 🤔 **什麼是 I2C 協定？**

I2C 協定是一種同步串列通訊標準，使多個裝置能夠透過共用的雙線匯流排進行通訊。它使用主從式架構，支援多個主控裝置，非常適合在嵌入式系統中連接多個積體電路、感測器和周邊裝置。

### **核心概念**

**雙線通訊：**
- **SDA 線路**：串列資料線路，用於雙向資料傳輸
- **SCL 線路**：串列時脈線路，用於同步
- **開汲極配置**：開汲極配置，用於線 AND 運算
- **上拉電阻**：外部上拉電阻，用於訊號位準

**主從式架構：**
- **主控裝置**：發起通訊並控制匯流排的裝置
- **從屬裝置**：回應主控裝置命令的裝置
- **多主控支援**：同一匯流排上支援多個主控裝置
- **動態角色指派**：動態角色指派與切換

**定址系統：**
- **7 位元定址**：標準 7 位元裝置定址
- **10 位元定址**：擴展 10 位元裝置定址
- **廣播定址**：對所有裝置進行廣播定址
- **位址指派**：位址指派與管理

**同步通訊：**
- **時脈驅動**：時脈驅動的資料傳輸與接收
- **同步化**：裝置間自動同步
- **時序控制**：精確的時序控制與管理
- **時脈延展**：為慢速裝置提供時脈延展

### **I2C 通訊流程**

**基本通訊過程：**
```
主控裝置                          從屬裝置
     │                            │
     │  ┌─────────┐              │
     │  │  資料   │              │
     │  │  來源   │              │
     │  └─────────┘              │
     │       │                   │
     │  ┌─────────┐              │
     │  │ I2C     │              │
     │  │ 主控    │              │
     │  └─────────┘              │
     │       │                   │
     │  ┌─────────┐              │
     │  │ SDA/SCL │ ────────────┼── I2C 匯流排
     │  │ 線路    │              │
     │  └─────────┘              │
     │                            │  ┌─────────┐
     │                            │  │ I2C     │
     │                            │  │ 從屬    │
     │                            │  └─────────┘
     │                            │       │
     │                            │  ┌─────────┐
     │                            │  │  資料   │
     │                            │  │  接收   │
     │                            │  └─────────┘
```

**匯流排拓撲：**
```
┌─────────────────────────────────────────────────────────────┐
│                    I2C 匯流排網路                            │
├─────────────────┬─────────────────┬─────────────────────────┤
│   主控 1        │   主控 2        │      主控 N             │
│                 │                 │                         │
│  ┌───────────┐  │  ┌───────────┐  │  ┌─────────────────────┐ │
│  │ I2C       │  │  │ I2C       │  │  │   I2C               │ │
│  │ 主控      │  │  │ 主控      │  │  │   主控              │ │
│  └───────────┘  │  └───────────┘  │  └─────────────────────┘ │
│        │        │        │        │           │              │
│        └────────┼────────┼────────┼───────────┘              │
│                 │        │        │                          │
│              SDA ────────┼─────── SDA                        │
│                          │                                   │
│              SCL ────────┼─────── SCL                        │
│                          │                                   │
│                 ┌────────┼────────┐                          │
│                 │ 從屬 1 │ 從屬 N │                          │
│                 │        │        │                          │
│                 └────────┼────────┘                          │
└──────────────────────────┼──────────────────────────────────┘
```

## 🎯 **為什麼 I2C 協定很重要？**

### **嵌入式系統需求**

**系統整合：**
- **多裝置支援**：單一匯流排上支援多個裝置
- **簡易接線**：多個裝置使用簡單的雙線連接
- **標準介面**：裝置通訊的標準介面
- **易於整合**：易於與現有系統整合

**效能與效率：**
- **高效通訊**：與多個裝置進行高效通訊
- **低額外負擔**：低協定額外負擔與複雜度
- **快速傳輸**：快速的資料傳輸與通訊
- **即時操作**：即時操作與回應

**可靠性與穩健性：**
- **ACK/NACK 交握**：位元組級確認，用於存在偵測與流量控制
- **時脈延展**：從屬裝置可在需要時減慢匯流排速度
- **仲裁**：主控裝置間的非破壞性仲裁
- **雜訊考量**：開汲極單端訊號；雜訊免疫力取決於上拉電阻、匯流排電容與速度

**成本與複雜度：**
- **低成本**：低成本的實作與元件
- **簡單設計**：簡單的設計與實作
- **標準元件**：標準元件與可取得性
- **易於除錯**：易於除錯與故障排除

### **實際影響**

- **資訊娛樂**：音訊、視訊與導航系統

**醫療裝置：**

## 💻 **實作**

### **基本 I2C 實作**

**I2C 配置：**
```c
// I2C 配置結構體
typedef struct {
    uint32_t clock_speed;      // 時脈速度（赫茲）
    uint8_t  addressing_mode;  // 7 位元或 10 位元定址
    uint8_t  dual_addressing;  // 雙位址支援
    uint8_t  general_call;     // 通用呼叫支援
    uint8_t  no_stretch;       // 停用時脈延展
} I2C_Config_t;

// 使用配置初始化 I2C
HAL_StatusTypeDef i2c_init(I2C_HandleTypeDef* hi2c, I2C_Config_t* config) {
    hi2c->Instance = I2C1;
    hi2c->Init.ClockSpeed = config->clock_speed;
    hi2c->Init.DutyCycle = I2C_DUTYCYCLE_2;
    hi2c->Init.OwnAddress1 = 0;
    hi2c->Init.AddressingMode = config->addressing_mode == 10 ? I2C_ADDRESSINGMODE_10BIT : I2C_ADDRESSINGMODE_7BIT;
    hi2c->Init.DualAddressMode = config->dual_addressing ? I2C_DUALADDRESS_ENABLE : I2C_DUALADDRESS_DISABLE;
    hi2c->Init.GeneralCallMode = config->general_call ? I2C_GENERALCALL_ENABLE : I2C_GENERALCALL_DISABLE;
    hi2c->Init.NoStretchMode = config->no_stretch ? I2C_NOSTRETCH_ENABLE : I2C_NOSTRETCH_DISABLE;
    
    return HAL_I2C_Init(hi2c);
}
```

**資料傳輸：**
```c
// 傳送 I2C 資料
HAL_StatusTypeDef i2c_transmit(I2C_HandleTypeDef* hi2c, uint16_t device_address, uint8_t* data, uint16_t size) {
    return HAL_I2C_Master_Transmit(hi2c, device_address, data, size, HAL_MAX_DELAY);
}

// 接收 I2C 資料
HAL_StatusTypeDef i2c_receive(I2C_HandleTypeDef* hi2c, uint16_t device_address, uint8_t* data, uint16_t size) {
    return HAL_I2C_Master_Receive(hi2c, device_address, data, size, HAL_MAX_DELAY);
}
```

## ❓ **面試問題**

1. **什麼是 I2C 協定，為什麼要使用它？**
   - I2C 是一種同步、多主控、雙線串列通訊協定
   - 用於積體電路與周邊裝置之間的通訊

2. **I2C 的主要特性有哪些？**
   - 雙線通訊（SDA、SCL）、多主控支援、定址系統
   - 時脈延展、仲裁與錯誤偵測

3. **I2C 定址如何運作？**
   - 7 位元或 10 位元裝置定址，附帶讀/寫位元
   - 多裝置的位址指派與管理

4. **什麼是 I2C 的時脈延展？**
   - 從屬裝置可以將 SCL 拉低以減慢通訊速度
   - 用於慢速裝置或處理時間需求

### **進階問題**

   - 時脈速度、定址、上拉電阻、時序需求
   - 硬體與軟體整合考量

3. **如何最佳化 I2C 效能？**
   - 最佳化時脈速度、降低匯流排電容、改善時序
   - 考慮系統需求與限制

4. **I2C 實作有哪些挑戰？**

## 🧪 **實作練習**

### **練習 1：I2C 位址掃描**
**目標**：探索匯流排上的所有 I2C 裝置。
**設置**：將多個 I2C 裝置連接到單一匯流排。
**步驟**：
1. 初始化 I2C 主控
2. 掃描所有可能的位址（0x08 到 0x77）
3. 傳送 START + 位址 + R/W 位元
4. 檢查 ACK 回應
5. 記錄所有回應的位址
**預期結果**：匯流排上所有活動 I2C 裝置的清單。

### **練習 2：時脈延展展示**
**目標**：觀察並處理時脈延展行為。
**設置**：使用支援時脈延展的 I2C 裝置（例如 EEPROM）。
**步驟**：
1. 以慢速時脈（100 kHz）配置 I2C
2. 傳送寫入命令
3. 在 ACK 期間監控 SCL 線路
4. 測量延展持續時間
5. 實作逾時處理
**預期結果**：了解裝置何時及為何延展時脈。

### **練習 3：多主控仲裁**
**目標**：展示 I2C 仲裁與碰撞偵測。
**設置**：同一匯流排上的兩個 I2C 主控。
**步驟**：
1. 以不同位址配置兩個主控
2. 啟動同時傳輸
3. 監控 SDA 線路的仲裁情況
4. 觀察哪個主控取得優先權
5. 處理碰撞偵測
**預期結果**：了解 I2C 如何處理多個主控。

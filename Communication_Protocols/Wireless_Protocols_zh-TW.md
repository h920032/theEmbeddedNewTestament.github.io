## 概念 → 重要性 → 最小範例 → 動手試試 → 重點摘要

**概念**：嵌入式系統中的無線協定是為您的特定應用需求選擇正確的通訊方式。每種協定在範圍、功耗、資料速率和可靠性之間提供不同的權衡，使協定選擇成為關鍵的設計決策。

**重要性**：無線連線對於現代嵌入式系統至關重要，可實現 IoT 裝置、穿戴式技術和遠端監控。選錯協定可能導致效能不佳、功耗過高或通訊不可靠，而正確的選擇則能實現新的應用並改善使用者體驗。

**最小範例**：一個 BLE 溫度感測器，展示透過週期性資料傳輸實現的超低功耗無線通訊。

**動手試試**：實作一個簡單的 WiFi 客戶端，連線到網路並傳送資料，觀察功耗和連線穩定性。

**重點摘要**：無線協定選擇需要了解您的應用特定需求、環境限制和功耗要求。最佳協定需在這些因素之間取得平衡，同時提供可靠、高效的通訊。

## 🎯 **概述**

無線協定是通訊標準，能夠在無需實體連線的情況下進行資料傳輸，使用電磁波在裝置之間傳遞資訊。這些協定對於需要行動性、靈活性和連線能力的現代嵌入式系統至關重要，應用包括 IoT 裝置、穿戴式技術、智慧家庭系統和工業無線網路。

### **關鍵概念**
- **無線通訊** - 不需要實體連線的資料傳輸
- **協定多樣性** - 為不同應用和需求提供多種協定
- **範圍與功耗** - 通訊範圍與功耗之間的權衡
- **干擾管理** - 處理無線干擾和雜訊
- **安全考量** - 無線安全和認證機制

## 🤔 **什麼是無線協定？**

無線協定是通訊標準，使裝置能夠透過電磁波交換資料，而無需實體連線。它們為嵌入式系統中的無線通訊提供基礎，在範圍、功耗、資料速率和可靠性之間提供各種權衡，以滿足不同的應用需求。

### **核心概念**

**無線通訊：**
- **電磁傳輸**：透過電磁波進行資料傳輸
- **頻段**：各種頻段和頻譜分配
- **調變技術**：數位調變和訊號處理
- **天線系統**：天線設計和輻射模式

**協定特性：**
- **範圍能力**：通訊範圍和覆蓋區域
- **功耗**：功耗和電池壽命
- **資料速率**：資料傳輸速率和吞吐量
- **可靠性**：通訊可靠性和錯誤處理

**系統整合：**
- **硬體整合**：無線硬體整合和支援
- **軟體整合**：無線軟體整合和管理
- **網路管理**：無線網路管理和控制
- **安全實作**：無線安全和認證

## 🔵 **Bluetooth 與 BLE**

### **Bluetooth 基礎**

**Bluetooth 架構：**
- **微微網**：Bluetooth 微微網和網路結構
- **主從式**：主從式通訊模型
- **裝置發現**：裝置發現和配對
- **服務發現**：服務發現和管理

**Bluetooth 版本：**
- **Bluetooth Classic**：用於高速資料的傳統 Bluetooth

**Bluetooth 特性：**
- **短距離**：短距離通訊（通常 10-100 公尺）
- **低功耗**：低功耗和電池壽命

## 💻 **實作**

### **基本無線協定實作**

**BLE 實作：**
```c
// BLE 裝置設定結構
typedef struct {
    uint8_t  device_name[16];      // 裝置名稱
    uint16_t appearance;            // 裝置外觀
    uint8_t  tx_power;             // 傳輸功率
    uint16_t min_conn_interval;    // 最小連線間隔
    uint16_t max_conn_interval;    // 最大連線間隔
    uint16_t slave_latency;        // 從屬延遲
    uint16_t supervision_timeout;  // 監督超時
} BLE_Device_Config_t;

// 初始化 BLE 裝置
BLE_Status_t ble_device_init(BLE_Device_Config_t* config) {
    ble_device_config = *config;
    
    // 初始化 BLE 硬體
    if (ble_hardware_init() != BLE_STATUS_SUCCESS) {
        return BLE_STATUS_ERROR;
    }
    
    // 初始化 BLE 協定堆疊
    if (ble_protocol_init() != BLE_STATUS_SUCCESS) {
        return BLE_STATUS_ERROR;
    }
    
    return BLE_STATUS_SUCCESS;
}
```

**WiFi 實作：**
```c
// WiFi 設定結構
typedef struct {
    char     ssid[32];             // 網路 SSID
    char     password[64];         // 網路密碼
    uint8_t  security_type;        // 安全類型（WPA、WPA2 等）
    uint8_t  channel;              // WiFi 頻道
    uint8_t  power_mode;           // 電源模式
    uint16_t beacon_interval;      // 信標間隔
} WiFi_Config_t;

// 初始化 WiFi
WiFi_Status_t wifi_init(WiFi_Config_t* config) {
    wifi_config = *config;
    
    // 初始化 WiFi 硬體
    if (wifi_hardware_init() != WIFI_STATUS_SUCCESS) {
        return WIFI_STATUS_ERROR;
    }
    
    // 初始化 WiFi 協定堆疊
    if (wifi_protocol_init() != WIFI_STATUS_SUCCESS) {
        return WIFI_STATUS_ERROR;
    }
    
    return WIFI_STATUS_SUCCESS;
}
```

## 實作實驗

### 實驗 1：BLE 功耗分析
**目標**：了解 BLE 通訊的功耗特性。

**設置**：實作一個具有可設定廣播和連線間隔的 BLE 溫度感測器。

**步驟**：
1. 使用不同的廣播間隔（100ms、500ms、1000ms）設定 BLE
2. 測量廣播和連線狀態下的電流消耗
3. 實作一個具有可設定更新速率的簡單溫度服務
4. 使用各種連線參數（最小/最大間隔、從屬延遲）進行測試
5. 計算不同運作條件下的電池壽命

**預期結果**：了解 BLE 參數如何影響功耗和電池壽命。

### 實驗 2：WiFi 連線穩定性測試
**目標**：評估各種網路條件下的 WiFi 連線穩定性。

**設置**：建立一個監控連線品質並自動重新連線的 WiFi 客戶端。

**步驟**：
1. 實作具有可設定重試參數的 WiFi 連線
2. 新增訊號強度監控和連線品質指標
3. 在不同訊號強度下測試連線穩定性
4. 實作具有指數退避的自動重新連線
5. 測量不同條件下的重新連線時間和成功率

**預期結果**：為嵌入式應用提供健全的 WiFi 連線管理。

### 實驗 3：協定選擇決策矩陣
**目標**：建立無線協定選擇的系統化方法。

**設置**：開發一個根據應用需求評估協定的決策矩陣工具。

**步驟**：
1. 定義評估標準（範圍、功耗、資料速率、成本、安全性）
2. 根據應用重要性為每個標準分配權重
3. 為每個協定（Bluetooth、BLE、WiFi、Zigbee、LoRa）在每個標準上評分
4. 計算加權分數並排列協定名次
5. 透過實際測試和量測驗證結果

**預期結果**：為嵌入式應用提供無線協定選擇的系統化方法。

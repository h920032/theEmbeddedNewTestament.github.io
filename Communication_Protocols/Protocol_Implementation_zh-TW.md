## 🎯 **概述**

協定實作是為嵌入式系統設計和實作自訂通訊協定的過程。它涉及建立結構化的資料格式、訊息框架、錯誤偵測機制和狀態管理，以確保裝置之間的可靠通訊。理解協定實作對於建立穩健、高效且可擴展的嵌入式通訊系統至關重要。

### **關鍵概念**
- **自訂協定設計** - 協定架構、訊息結構和通訊流程
- **訊息框架** - 起始/結束定界符、長度欄位和框架同步
- **錯誤偵測** - 校驗和、CRC 和錯誤更正機制
- **狀態管理** - 協定狀態、轉換和錯誤處理
- **效能最佳化** - 協定效率、頻寬利用和延遲

### **面試官意圖（他們在探查什麼）**
- 你能設計一個明確且可測試的協定嗎？
- 你是否理解框架、重新同步和錯誤恢復？
- 你能建構一個簡單的狀態機並為其邊界情況辯護嗎？

## 🧠 **概念優先**

### **協定 vs 實作**
**概念**：協定是規格，實作是實際遵循規格的程式碼。
**為什麼重要**：理解這個區別有助於你設計可實作的協定和可維護的實作。
**最小範例**：設計一個簡單的協定規格，然後用 C 語言實作它。
**試試看**：建立一份協定規格文件，然後實作它。
**要點**：好的協定是清晰、完整且可測試的。

### **狀態機複雜度**
**概念**：協定狀態機的範圍可以從簡單到複雜，影響可靠性和除錯。
**為什麼重要**：複雜的狀態機更難除錯，更容易出現邊界情況失敗。
**最小範例**：比較一個簡單的請求-回應協定和一個複雜的多階段協定。
**試試看**：實作兩者並測量除錯時間和可靠性。
**要點**：較簡單的協定通常更可靠且更容易維護。

## 🤔 **什麼是協定實作？**

協定實作是設計、開發和部署自訂通訊協定的系統化過程，使嵌入式裝置之間能夠進行可靠的資料交換。它涵蓋協定規格、訊息格式、錯誤處理機制和狀態管理系統的建立，以確保穩健且高效的通訊。

### **核心概念**

**協定架構：**
- **協定堆疊**：分層協定架構與設計
- **訊息結構**：結構化的訊息格式與組織
- **通訊流程**：資料流與通訊模式
- **錯誤處理**：錯誤偵測、更正與恢復機制

**協定設計：**
- **需求分析**：協定需求與規格
- **架構設計**：協定架構與結構
- **介面設計**：協定介面與 API 設計
- **實作策略**：實作方法與方法論

**協定管理：**
- **狀態管理**：協定狀態與轉換管理
- **資源管理**：資源分配與管理
- **效能管理**：效能監控與最佳化
- **安全管理**：安全性與存取控制

## 💻 **實作**

### **基本協定實作**

**協定結構：**
```c
// 協定訊息結構
typedef struct {
    uint8_t  start_delimiter;    // 起始定界符
    uint8_t  message_type;       // 訊息類型識別碼
    uint16_t message_length;     // 訊息長度
    uint8_t  source_address;     // 來源裝置位址
    uint8_t  destination_address; // 目的裝置位址
    uint8_t* data;               // 訊息資料承載
    uint16_t checksum;           // 訊息校驗和
    uint8_t  end_delimiter;      // 結束定界符
} Protocol_Message_t;

// 協定狀態列舉
typedef enum {
    PROTOCOL_STATE_IDLE,         // 閒置狀態
    PROTOCOL_STATE_RECEIVING,    // 接收訊息中
    PROTOCOL_STATE_PROCESSING,   // 處理訊息中
    PROTOCOL_STATE_SENDING,      // 傳送訊息中
    PROTOCOL_STATE_ERROR         // 錯誤狀態
} Protocol_State_t;

// 協定組態結構
typedef struct {
    uint8_t  device_address;     // 裝置位址
    uint32_t timeout_ms;         // 逾時時間（毫秒）
    uint16_t max_message_length; // 最大訊息長度
    uint8_t  retry_count;        // 失敗訊息的重試次數
} Protocol_Config_t;
```

**協定實作：**
```c
// 初始化協定
Protocol_Status_t protocol_init(Protocol_Config_t* config) {
    protocol_config = *config;
    protocol_state = PROTOCOL_STATE_IDLE;
    protocol_buffer = malloc(config->max_message_length);
    
    if (protocol_buffer == NULL) {
        return PROTOCOL_STATUS_ERROR;
    }
    
    return PROTOCOL_STATUS_SUCCESS;
}

// 傳送協定訊息
Protocol_Status_t protocol_send_message(Protocol_Message_t* message) {
    if (protocol_state != PROTOCOL_STATE_IDLE) {
        return PROTOCOL_STATUS_BUSY;
    }
    
    protocol_state = PROTOCOL_STATE_SENDING;
    
    // 傳送起始定界符
    uart_transmit_byte(message->start_delimiter);
    
    // 傳送訊息標頭
    uart_transmit_byte(message->message_type);
    uart_transmit_byte((message->message_length >> 8) & 0xFF);
    uart_transmit_byte(message->message_length & 0xFF);
    uart_transmit_byte(message->source_address);
    uart_transmit_byte(message->destination_address);
    
    // 傳送訊息資料
    for (uint16_t i = 0; i < message->message_length; i++) {
        uart_transmit_byte(message->data[i]);
    }
    
    // 傳送校驗和
    uart_transmit_byte((message->checksum >> 8) & 0xFF);
    uart_transmit_byte(message->checksum & 0xFF);
    
    // 傳送結束定界符
    uart_transmit_byte(message->end_delimiter);
    
    protocol_state = PROTOCOL_STATE_IDLE;
    return PROTOCOL_STATUS_SUCCESS;
}
```

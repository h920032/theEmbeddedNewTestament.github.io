## 🎯 **概述**

RS232、RS422 和 RS485 是串列通訊標準，定義了資料傳輸的電氣特性、訊號位準及通訊協定。這些標準廣泛應用於工業、汽車及嵌入式系統中，以在各種距離和環境下實現可靠的資料通訊。

### **核心概念**
- **電氣標準** - 訊號位準、電壓範圍及電氣特性
- **多站通訊** - 支援單一匯流排上的多個裝置
- **抗雜訊能力** - 差動訊號提高抗雜訊性
- **距離限制** - 纜線長度與速度的權衡
- **驅動器/接收器相容性** - 硬體介面需求

### **面試官意圖（他們在探測什麼）**
- 你能否根據距離/雜訊/拓撲選擇正確的標準？
- 你是否理解單端訊號與差動訊號的差異？
- 你能否推理終端匹配和匯流排負載？

## 🧠 **概念優先**

### **單端訊號 vs 差動訊號**
**概念**：RS232 使用單端訊號，而 RS422/RS485 使用差動訊號。
**重要性**：差動訊號提供更好的抗雜訊能力，允許更長的纜線距離，這對工業和汽車應用至關重要。
**最小範例**：比較單端訊號與差動訊號在 100 公尺纜線上的訊號完整性。
**動手試試**：使用示波器測量單端訊號與差動訊號上的雜訊。
**要點**：差動訊號以複雜度換取穩健性和距離。

### **多站 vs 點對點**
**概念**：RS232 是點對點的，RS422 是點對點差動的，RS485 支援單一匯流排上的多個裝置。
**重要性**：多站能力允許用較少的線路建構複雜網路，這對工業控制系統至關重要。
**最小範例**：使用不同標準為 8 個感測器設計網路拓撲。
**動手試試**：使用 RS485 實作一個簡單的多站網路。
**要點**：根據你的網路拓撲需求選擇標準。

## 🤔 **什麼是串列通訊標準？**

串列通訊標準是定義電子裝置之間可靠資料傳輸所需的電氣特性、訊號位準、時序及協定需求的規範。這些標準確保不同製造商和應用之間的相容性、互通性及可靠通訊。

### **核心概念**

**電氣標準：**
- **訊號位準**：定義邏輯高位準和低位準的電壓值
- **時序需求**：資料傳輸的精確時序需求
- **抗雜訊能力**：抗雜訊能力及訊號完整性規格
- **距離限制**：最大可靠通訊距離

**協定標準：**
- **資料格式**：標準化的資料格式和封框
- **錯誤偵測**：錯誤偵測和糾正機制
- **流量控制**：流量控制和交握協定
- **相容性**：相容性和互通性需求

**介面標準：**
- **連接器類型**：標準化的連接器類型和接腳配置
- **纜線規格**：纜線規格和需求
- **驅動器/接收器**：驅動器和接收器規格
- **終端匹配**：終端匹配和阻抗匹配需求

### **標準演進**

**歷史發展：**
- **RS232（1960年代）**：最初的串列通訊標準
- **RS422（1975年）**：差動訊號以改善效能
- **RS485（1983年）**：多站通訊能力
- **現代標準**：現代應用的演進和適應

**標準特性：**
- **向後相容性**：與舊標準的向後相容性
- **效能改進**：隨時間的效能改進
- **應用特定**：針對特定應用的適應和擴展
- **產業採用**：產業採用和標準化

## 💻 **實作**

### **基本 RS232 實作**

**RS232 配置：**
```c
// RS232 配置結構
typedef struct {
    uint32_t baud_rate;         // 鮑率（通常 9600-115200）
    uint8_t  data_bits;         // 資料位元（7, 8）
    uint8_t  stop_bits;         // 停止位元（1, 2）
    uint8_t  parity;            // 同位檢查（NONE, EVEN, ODD）
    uint8_t  flow_control;      // 流量控制（NONE, RTS_CTS）
} RS232_Config_t;

// 初始化 RS232 通訊
HAL_StatusTypeDef rs232_init(RS232_HandleTypeDef* hrs232, RS232_Config_t* config) {
    hrs232->Init.BaudRate = config->baud_rate;
    hrs232->Init.WordLength = config->data_bits == 8 ? UART_WORDLENGTH_8B : UART_WORDLENGTH_7B;
    hrs232->Init.StopBits = config->stop_bits == 2 ? UART_STOPBITS_2 : UART_STOPBITS_1;
    hrs232->Init.Parity = config->parity;
    hrs232->Init.Mode = UART_MODE_TX_RX;
    hrs232->Init.HwFlowCtl = config->flow_control;
    hrs232->Init.OverSampling = UART_OVERSAMPLING_16;
    
    return HAL_UART_Init(hrs232);
}
```

**RS422/RS485 配置：**
```c
// RS422/RS485 配置結構
typedef struct {
    uint32_t baud_rate;         // 鮑率
    uint8_t  data_bits;         // 資料位元
    uint8_t  stop_bits;         // 停止位元
    uint8_t  parity;            // 同位檢查
    uint8_t  mode;              // RS422 或 RS485 模式
    uint8_t  termination;       // 終端匹配啟用
} RS422_485_Config_t;

// 初始化 RS422/RS485 通訊
HAL_StatusTypeDef rs422_485_init(RS422_485_HandleTypeDef* hrs422_485, RS422_485_Config_t* config) {
    hrs422_485->Init.BaudRate = config->baud_rate;
    hrs422_485->Init.WordLength = config->data_bits == 8 ? UART_WORDLENGTH_8B : UART_WORDLENGTH_7B;
    hrs422_485->Init.StopBits = config->stop_bits == 2 ? UART_STOPBITS_2 : UART_STOPBITS_1;
    hrs422_485->Init.Parity = config->parity;
    hrs422_485->Init.Mode = config->mode;
    hrs422_485->Init.Termination = config->termination;
    
    return HAL_UART_Init(hrs422_485);
}
```

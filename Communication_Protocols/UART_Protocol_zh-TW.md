## 🎯 **概述**

UART（通用非同步收發器）是嵌入式系統中廣泛使用的串列通訊協定。它提供簡單、可靠且具成本效益的裝置間通訊，無需共用時脈訊號，使其成為嵌入式應用中點對點通訊的理想選擇。

### **核心概念**
- **非同步通訊**：不需要共用時脈訊號
- **鮑率**：以每秒位元數表示的資料傳輸速率
- **資料框架**：起始位元、資料位元、同位元、停止位元
- **錯誤偵測**：同位元檢查、框架錯誤、溢位偵測
- **流量控制**：硬體與軟體流量控制機制

## 🧠 **概念先行**

### **非同步 vs 同步**
**概念**：UART 使用起始/停止位元而非共用時脈進行同步。
**重要性**：這使得 UART 實作簡單，但需要精確的時序和鮑率匹配。
**最小範例**：實作一個帶有起始/停止位元的基本 UART 發送器。
**動手試試**：建立一個簡單的類 UART 協定並測試時序容許誤差。
**重點摘要**：UART 簡單但對時序要求嚴格；非常適合簡潔性比速度更重要的嵌入式系統。

### **鮑率與時序**
**概念**：鮑率同時決定資料速率和時序精確度要求。
**重要性**：較高的鮑率意味著更快的通訊，但需要更精確的時序和更好的訊號完整性。
**最小範例**：計算不同鮑率和資料框架大小的時序。
**動手試試**：使用示波器在不同鮑率下測量實際時序。
**重點摘要**：根據你的時序精確度和訊號品質能力來選擇鮑率。

## 🤔 **什麼是 UART 協定？**

UART 協定是一種非同步串列通訊標準，使裝置間的資料傳輸無需共用時脈訊號。它使用預定義的鮑率和資料框架結構來確保可靠通訊，是嵌入式系統中最基本且最廣泛使用的通訊協定之一。

### **核心概念**

**非同步通訊：**
- **無共用時脈**：裝置間不需要共用時脈訊號
- **鮑率協議**：透過鮑率協議進行同步
- **起始/停止位元**：使用起始和停止位元進行框架同步
- **時序容許誤差**：時序容許誤差與彈性

**資料傳輸：**
- **串列傳輸**：資料位元的循序傳輸
- **基於框架**：資料以特定結構組織成框架
- **雙向傳輸**：支援雙向通訊
- **即時性**：即時資料傳輸能力

**錯誤偵測：**
- **同位元檢查**：用於錯誤偵測的同位元檢查
- **框架錯誤**：框架錯誤偵測與處理
- **溢位偵測**：緩衝區溢位偵測與預防
- **錯誤恢復**：錯誤恢復與重傳機制

## 💻 **實作範例**

### **基本 UART 實作**

**UART 設定：**
```c
// UART 設定結構
typedef struct {
    uint32_t baud_rate;      // 鮑率（每秒位元數）
    uint32_t data_bits;      // 資料位元數（7、8、9）
    uint32_t stop_bits;      // 停止位元數（1、2）
    uint32_t parity;         // 同位元類型（NONE、EVEN、ODD）
    uint32_t flow_control;   // 流量控制（NONE、RTS_CTS、RTS_CTS_DTR_DSR）
    uint32_t mode;           // 模式（RX_ONLY、TX_ONLY、TX_RX）
} UART_Config_t;

// 使用設定初始化 UART
HAL_StatusTypeDef uart_init(UART_HandleTypeDef* huart, UART_Config_t* config) {
    huart->Instance = USART1;
    huart->Init.BaudRate = config->baud_rate;
    huart->Init.WordLength = config->data_bits == 9 ? UART_WORDLENGTH_9B : UART_WORDLENGTH_8B;
    huart->Init.StopBits = config->stop_bits == 2 ? UART_STOPBITS_2 : UART_STOPBITS_1;
    huart->Init.Parity = config->parity;
    huart->Init.Mode = config->mode;
    huart->Init.HwFlowCtl = config->flow_control;
    huart->Init.OverSampling = UART_OVERSAMPLING_16;
    
    return HAL_UART_Init(huart);
}
```

**資料傳輸：**
```c
// 傳送 UART 資料
HAL_StatusTypeDef uart_transmit(UART_HandleTypeDef* huart, uint8_t* data, uint16_t size) {
    return HAL_UART_Transmit(huart, data, size, HAL_MAX_DELAY);
}

// 接收 UART 資料
HAL_StatusTypeDef uart_receive(UART_HandleTypeDef* huart, uint8_t* data, uint16_t size) {
    return HAL_UART_Receive(huart, data, size, HAL_MAX_DELAY);
}
```

## ⚠️ **常見陷阱**

### **設定錯誤**

- **症狀**：接收到亂碼或不正確的資料
- **原因**：裝置之間鮑率不匹配
- **解決方案**：確保相同的鮑率設定
- **預防**：使用標準鮑率並驗證設定

**資料格式不匹配：**
- **症狀**：資料解讀不正確或框架錯誤
- **原因**：資料位元、同位元或停止位元不匹配
- **解決方案**：確保相同的資料格式設定
- **預防**：記錄並驗證資料格式要求

**流量控制問題：**
- **症狀**：資料遺失或通訊停滯

- **預防**：監控緩衝區使用量並實作溢位保護

**中斷處理問題：**
- **症狀**：漏接資料或系統不穩定
- **原因**：中斷處理不當或優先權問題

- **預防**：遵循中斷處理最佳實踐

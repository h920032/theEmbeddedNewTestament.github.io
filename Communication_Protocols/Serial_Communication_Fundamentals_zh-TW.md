## 🎯 **概述**

串列通訊是一種基礎的資料傳輸方法，資訊透過通訊通道一次傳送一個位元。它是大多數嵌入式通訊協定的基礎，對於理解 UART、SPI、I2C 及其他現代嵌入式系統中使用的串列協定至關重要。

### **核心概念**
- **逐位元傳輸** - 資料隨時間依序傳送
- **傳輸模式** - 單工、半雙工及全雙工
- **同步** - 基於時脈與非同步方法
- **資料框架** - 起始位元、資料位元、同位檢查、停止位元
- **錯誤偵測** - 同位檢查、校驗和、CRC

- 你能比較非同步與同步及其取捨嗎？
- 你了解框架與錯誤偵測的基礎嗎？

## 🧠 **概念優先**

### **為什麼是串列而非並列？**
**概念**：串列通訊以頻寬換取可靠性與距離。
**重要性**：在嵌入式系統中，你經常需要透過較長距離或噪音環境進行通訊，而並列訊號在這些情境下並不實際。
**最小範例**：比較 8 位元並列與串列傳輸在 1 公尺導線上的表現。
**試一試**：測量並列與串列在不同距離下的訊號完整性。
**重點**：串列對嵌入式應用更為穩健，即使每位元速度較慢。

### **同步策略**
**概念**：基於時脈（同步）與自時脈（非同步）方法。
**重要性**：不同的同步策略在嵌入式系統中有不同的取捨。
**最小範例**：實作一個類似 UART 的簡單協定，使用起始/停止位元。
**試一試**：建立一個基本的串列傳送器和接收器，搭配 LED 指示燈。
**重點**：非同步較簡單但需要精確的時序；同步較複雜但效率更高。

## 🤔 **什麼是串列通訊？**

串列通訊是一種資料傳輸方法，數位資訊透過單一通訊通道依序一次傳送一個位元來傳遞。不同於同時傳送多個位元的並列通訊，串列通訊使用分時多工來依序傳輸資料，使其更適合長距離通訊和易受噪音干擾的環境。

### **核心概念**

**依序資料傳輸：**
- **逐位元傳輸**：資料一次傳輸一個位元
- **分時**：基於時間的資料組織與傳輸
- **依序處理**：依序處理資料位元
- **時間組織**：資料傳輸的時間組織

**通訊通道：**
- **單一通道**：用於資料傳輸的單一通訊通道
- **雙向能力**：支援雙向通訊
- **通道特性**：通道特性與限制
- **訊號完整性**：訊號完整性與品質維護

**資料組織：**
- **資料框架**：將資料組織成訊框或封包
- **同步**：傳送器與接收器之間的同步
- **錯誤控制**：錯誤偵測與修正機制
- **流量控制**：流量控制與資料速率管理

**基本傳輸過程：**

     │  ┌─────────┐              │
     │  │  資料   │              │
     │  │  來源   │              │
     │  └─────────┘              │
     │       │                   │
     │  ┌─────────┐              │
     │  │ 並列    │              │
     │  │ 轉      │              │
     │  │ 串列    │              │
     │  └─────────┘              │
     │       │                   │
     │  ┌─────────┐              │
     │  │ 串列    │ ────────────┼── 通訊通道
     │  │ 資料    │              │
     │  └─────────┘              │
     │                            │  ┌─────────┐
     │                            │  │ 串列    │
     │                            │  │ 轉      │
     │                            │  │ 並列    │
     │                            │  └─────────┘

**資料流特性：**
- **傳輸路徑**：並列資料 → 串列轉換 → 通道 → 串列轉並列轉換
- **時序控制**：精確的資料傳輸時序控制
- **同步**：傳送器與接收器之間的同步

## 💻 **實作**

### **基本串列通訊**

**串列組態：**
```c
// 串列通訊組態
typedef struct {
    uint32_t baud_rate;         // 每秒位元數
    uint8_t  data_bits;         // 資料位元數
    uint8_t  stop_bits;         // 停止位元數
    uint8_t  parity;            // 同位類型
    uint8_t  flow_control;      // 流量控制方法
} Serial_Config_t;

// 初始化串列通訊
HAL_StatusTypeDef serial_init(Serial_HandleTypeDef* hserial, Serial_Config_t* config) {
    hserial->Init.BaudRate = config->baud_rate;
    hserial->Init.WordLength = config->data_bits == 9 ? UART_WORDLENGTH_9B : UART_WORDLENGTH_8B;
    hserial->Init.StopBits = config->stop_bits == 2 ? UART_STOPBITS_2 : UART_STOPBITS_1;
    hserial->Init.Parity = config->parity;
    hserial->Init.Mode = UART_MODE_TX_RX;
    hserial->Init.HwFlowCtl = config->flow_control;
    hserial->Init.OverSampling = UART_OVERSAMPLING_16;
    
    return HAL_UART_Init(hserial);
}
```

**資料傳輸：**
```c
// 傳送串列資料
HAL_StatusTypeDef serial_transmit(Serial_HandleTypeDef* hserial, uint8_t* data, uint16_t size) {
    return HAL_UART_Transmit(hserial, data, size, HAL_MAX_DELAY);
}

// 接收串列資料
HAL_StatusTypeDef serial_receive(Serial_HandleTypeDef* hserial, uint8_t* data, uint16_t size) {
    return HAL_UART_Receive(hserial, data, size, HAL_MAX_DELAY);
}
```

## ❓ **面試問題**

### **基礎問題**

1. **什麼是串列通訊，為什麼使用它？**
   - 串列通訊透過單一通道一次傳輸一個位元的資料
   - 用於可靠的長距離通訊，減少佈線需求

2. **串列通訊與並列通訊的主要差異是什麼？**
   - 串列：一次一個位元，較少導線，更適合長距離
   - 並列：一次多個位元，較多導線，更適合短距離

3. **串列通訊中有哪些不同的傳輸模式？**
   - 單工：單向通訊
   - 半雙工：雙向交替通訊
   - 全雙工：雙向同時通訊

4. **串列通訊中的錯誤偵測如何運作？**
   - 同位檢查、校驗和和 CRC 用於錯誤偵測
   - 各種錯誤修正方法用於錯誤恢復

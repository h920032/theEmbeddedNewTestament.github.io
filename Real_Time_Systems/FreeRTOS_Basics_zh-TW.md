## **概念 → 重要性 → 最小範例 → 動手試 → 重點整理**

**概念**：FreeRTOS 是一個即時作業系統，透過分配 CPU 時間片段給每個任務來管理多個任務，確保關鍵操作能在需要時準時完成。

**重要性**：沒有 RTOS，你必須手動管理程式不同部分之間的時序、優先權和資源共享。隨著複雜度增長，這會變得無法維護，且關鍵操作可能錯過其截止時間。

**最小範例**：一個簡單的系統，包含兩個任務——一個每 100ms 閃爍 LED，另一個每 500ms 讀取感測器。FreeRTOS 確保兩者都能可靠地運行，互不干擾。

**動手試**：從一個閃爍 LED 的單一任務開始，然後新增一個讀取感測器的第二個任務。觀察 FreeRTOS 如何自動管理兩者。

**重點整理**：FreeRTOS 提供可預測的時序和資源管理，讓你能專注於系統應該做什麼，而非如何協調多個操作。

## 📋 **快速參考：關鍵知識**

### **FreeRTOS 基本原理**
- **即時性**：保證可預測的時序，不一定是快速執行
- **搶佔式**：較高優先權的任務可以中斷較低優先權的任務
- **可移植**：可在多種微控制器架構上運作
- **開源**：MIT 授權，無需版稅或授權費用
- **可擴展**：可為需求最小或複雜的系統進行組態設定

### **核心元件**
- **排程器**：根據優先權管理哪個任務何時執行
- **任務管理器**：處理任務的建立、刪除和狀態轉換
- **記憶體管理器**：管理堆疊配置和記憶體池
- **計時服務**：提供延遲、逾時和週期性執行
- **通訊機制**：佇列、號誌和互斥鎖用於任務協調

### **任務狀態**
- **已建立**：任務存在但尚未排程
- **就緒**：任務已準備好執行，等待 CPU 時間
- **執行中**：任務目前正在 CPU 上執行
- **阻塞**：任務正在等待某些事物（延遲、資料、資源）
- **已刪除**：任務已從系統中移除

### **關鍵組態選項**
- **configUSE_PREEMPTION**：啟用/停用搶佔式排程
- **configTICK_RATE_HZ**：系統滴答頻率（通常為 1000Hz）
- **configMAX_PRIORITIES**：任務優先權的最大數量
- **configMINIMAL_STACK_SIZE**：任務的最小堆疊大小
- **configUSE_MUTEXES**：啟用互斥鎖支援以保護資源

## 🧠 **核心概念**

### **什麼是即時？**

即時並不代表「快速」——它代表**可預測**。即時系統保證操作在其指定的時間限制內完成。

```
┌─────────────────────────────────────────────────────────────┐
│                    即時 vs 非即時                           │
├─────────────────────────────────────────────────────────────┤
│                即時系統                                     │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐      │
│  │ 任務 A  │  │ 任務 B  │  │ 任務 A  │  │ 任務 B  │      │
│  │ 100ms   │  │ 500ms   │  │ 100ms   │  │ 500ms   │      │
│  └─────────┘  └─────────┘  └─────────┘  └─────────┘      │
│  ↑           ↑           ↑           ↑                    │
│  0ms        100ms       200ms       300ms                 │
│                                                           │
│                非即時系統                                   │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐      │
│  │ 任務 A  │  │ 任務 B  │  │ 任務 A  │  │ 任務 B  │      │
│  │ 100ms   │  │ 500ms   │  │ 150ms   │  │ 600ms   │      │
│  └─────────┘  └─────────┘  └─────────┘  └─────────┘      │
│  ↑           ↑           ↑           ↑                    │
│  0ms        100ms       250ms       350ms                 │
│                                                           │
│  ❌ 時序不固定——不可預測！                                 │
└─────────────────────────────────────────────────────────────┘
```

### **為什麼使用 RTOS 而非裸機？**

**裸機方式：**
```
┌─────────────────────────────────────────────────────────────┐
│                    裸機程式設計                             │
├─────────────────────────────────────────────────────────────┤
│  while(1) {                                               │
│    // 檢查是否到了閃爍 LED 的時間                           │
│    if (timer_elapsed(100ms)) {                            │
│      toggle_led();                                         │
│      reset_timer();                                        │
│    }                                                       │
│                                                            │
│    // 檢查是否到了讀取感測器的時間                           │
│    if (timer_elapsed(500ms)) {                            │
│      sensor_value = read_sensor();                         │
│      reset_timer();                                        │
│    }                                                       │
│                                                            │
│    // 如果需要新增更多任務怎麼辦？                          │
│    // 如果優先權改變怎麼辦？                                │
│    // 如果時序需求改變怎麼辦？                              │
│  }                                                         │
│                                                            │
│  ❌ 很快就變得無法維護！                                    │
└─────────────────────────────────────────────────────────────┘
```

**FreeRTOS 方式：**
```
┌─────────────────────────────────────────────────────────────┐
│                    FreeRTOS 方式                            │
├─────────────────────────────────────────────────────────────┤
│  // 任務 1：每 100ms 閃爍 LED                              │
│  void vBlinkTask(void *pvParameters) {                    │
│    while(1) {                                             │
│      toggle_led();                                         │
│      vTaskDelay(pdMS_TO_TICKS(100));                      │
│    }                                                       │
│  }                                                         │
│                                                            │
│  // 任務 2：每 500ms 讀取感測器                             │
│  void vSensorTask(void *pvParameters) {                    │
│    while(1) {                                             │
│      sensor_value = read_sensor();                         │
│      vTaskDelay(pdMS_TO_TICKS(500));                      │
│    }                                                       │
│  }                                                         │
│                                                            │
│  // FreeRTOS 自動處理剩下的一切！                           │
│  ✅ 簡潔、可維護、可擴展                                   │
└─────────────────────────────────────────────────────────────┘
```

## 📋 **任務管理**

### **什麼是任務？**

│                                                           │
│  • 已建立：任務存在但尚未排程                              │
│  • 就緒：任務已準備好執行，等待 CPU                        │

### **建立你的第一個任務**

以下是建立簡單任務的最小程式碼：

```c
#include "FreeRTOS.h"
#include "task.h"

// 任務函式——這是任務要做的事情
void vBlinkTask(void *pvParameters) {
    while (1) {
        // 切換 LED（在此放入你的硬體特定程式碼）
        toggle_led();
        
        // 等待 500ms——FreeRTOS 處理時序
        vTaskDelay(pdMS_TO_TICKS(500));
    }
}

// 在你的 main 函式中
int main(void) {
    // 建立任務
    xTaskCreate(
        vBlinkTask,        // 要執行的函式
        "BlinkTask",       // 任務名稱（用於除錯）
        128,               // 堆疊大小（以字組為單位）
        NULL,              // 參數（此處無）
        1,                 // 優先權（1 = 最低）
        NULL               // 任務控制代碼（此處不需要）
    );
    
    // 啟動 FreeRTOS 排程器
    vTaskStartScheduler();
    
    // 不應該執行到這裡
    while (1);
}
```

**重點：**
- `vTaskDelay()` 不會阻塞 CPU——它讓其他任務得以執行
- 堆疊大小（128）對簡單任務應已足夠
- 優先權 1 是最低優先權
- 任務在 `while(1)` 迴圈中永久執行

## 🔗 **同步機制**

### **為什麼任務需要同步？**

當多個任務共享資源（如感測器或通訊匯流排）時，它們需要協調：

```
┌─────────────────────────────────────────────────────────────┐
│                問題：資源衝突                                │
├─────────────────────────────────────────────────────────────┤
│                                                           │
│  任務 A：「我要讀取溫度感測器」                             │
│  任務 B：「我要讀取溫度感測器」                             │
│                                                           │
│  ❌ 兩者同時嘗試讀取 → 資料損毀！                          │
│                                                           │
│  解決方案：使用互斥鎖（mutual exclusion）                   │
│                                                           │
│  任務 A：「我先取得互斥鎖」                                │
│  任務 B：「我等待互斥鎖」                                  │
│                                                           │
│  ✅ 同一時間只有一個任務可以存取感測器                      │
└─────────────────────────────────────────────────────────────┘
```

### **使用互斥鎖進行基本同步**

```c
#include "FreeRTOS.h"
#include "task.h"
#include "semphr.h"

// 用於保護感測器的互斥鎖
SemaphoreHandle_t xSensorMutex;

// 讀取感測器的任務
void vSensorTask(void *pvParameters) {
    while (1) {
        // 等待互斥鎖（如有需要則無限等待）
        if (xSemaphoreTake(xSensorMutex, portMAX_DELAY) == pdTRUE) {
            // 已取得互斥鎖——可以安全讀取感測器
            float temperature = read_temperature_sensor();
            
            // 處理溫度資料
            process_temperature(temperature);
            
            // 歸還互斥鎖，使其他任務可以使用
            xSemaphoreGive(xSensorMutex);
        }
        
        // 下次讀取前等待
        vTaskDelay(pdMS_TO_TICKS(1000));
    }
}

// 初始化互斥鎖
void vInitializeSystem(void) {
    // 建立互斥鎖
    xSensorMutex = xSemaphoreCreateMutex();
    
    // 建立任務
    xTaskCreate(vSensorTask, "Sensor", 128, NULL, 2, NULL);
}
```

## ⚙️ **組態設定**

### **必要的組態選項**

FreeRTOS 具有高度可設定性。以下是你需要了解的關鍵設定：

```c
// FreeRTOSConfig.h - 關鍵組態選項

// 啟用搶佔式排程（任務可以互相中斷）
#define configUSE_PREEMPTION                    1

// 系統滴答頻率（FreeRTOS 多久檢查一次任務切換）
#define configTICK_RATE_HZ                      1000

// 任務優先權的最大數量
#define configMAX_PRIORITIES                    32

// 任務的最小堆疊大小
#define configMINIMAL_STACK_SIZE                128

// 啟用互斥鎖支援
#define configUSE_MUTEXES                       1

// 啟用佇列支援
#define configUSE_QUEUES                        1

// 啟用計數號誌支援
#define configUSE_COUNTING_SEMAPHORES           1

// 檢查堆疊溢位（對除錯很重要）
#define configCHECK_FOR_STACK_OVERFLOW          2
```

**這些設定的意義：**
- **搶佔式**：較高優先權的任務可以中斷較低優先權的任務
- **滴答頻率**：FreeRTOS 多久做一次排程決定
- **優先權數量**：可以使用多少種不同的優先權等級
- **堆疊大小**：每個任務獲得的最小記憶體
- **功能啟用**：啟用哪些 FreeRTOS 功能

## 🧪 **引導式實驗**

### **實驗 1：單一任務系統**
**目標**：理解基本的任務建立和時序控制。

**設定**：建立一個以特定頻率閃爍 LED 的單一任務。

**步驟**：
1. 建立一個切換 LED 的任務函式
2. 使用 `vTaskDelay()` 控制時序
3. 觀察一致的時序行為
4. 使用示波器測量實際時序（如果有的話）

**預期結果**：理解 FreeRTOS 提供可預測的時序。

### **實驗 2：多任務協調**
**目標**：學習任務如何協同工作。

**設定**：建立兩個任務——一個每 200ms 閃爍 LED A，另一個每 300ms 閃爍 LED B。

**步驟**：
1. 建立兩個獨立的任務函式
2. 給予它們不同的優先權
3. 觀察兩者如何獨立運行
4. 注意兩者的時序保持一致

**預期結果**：理解多個任務可以同時運行而互不干擾。

### **實驗 3：資源共享**
**目標**：學習同步和資源保護。

**設定**：建立兩個需要共享資源（如 UART 或感測器）的任務。

**步驟**：
1. 建立一個互斥鎖來保護共享資源
2. 讓兩個任務都嘗試使用該資源
3. 觀察互斥鎖如何防止衝突
4. 測量對時序的影響

**預期結果**：理解為什麼同步是必要的以及如何實作它。

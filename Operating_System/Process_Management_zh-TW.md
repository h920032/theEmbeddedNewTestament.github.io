## 🏗️ **程序基礎**

### **什麼是程序？**

程序是 Linux 中基本的執行單位——它是一個正在執行的程式實例，擁有自己的記憶體空間、執行上下文和系統資源。你可以將程序想像成一個「容器」，裡面包含了執行程式所需的一切：程式碼、資料、堆疊、堆積、檔案描述符等。

**程序抽象化：**

- **隔離性**：每個程序在自己的虛擬位址空間中執行
- **資源**：程序擁有專屬的系統資源（CPU 時間、記憶體、I/O）
- **身份識別**：每個程序都有一個唯一的程序識別碼（PID）
- **狀態**：程序可以處於各種狀態（執行中、等待中、停止）
- **階層結構**：程序形成具有父子關係的樹狀結構

#### **程序與程式：理解兩者的區別**

程式與程序之間的關係是理解 Linux 運作方式的基礎：

**程式（靜態）：**
- **定義**：包含可執行程式碼和資料的檔案
- **儲存**：以二進位檔案形式儲存在磁碟上
- **內容**：機器指令、靜態資料、符號表
- **狀態**：在被執行之前處於非活動狀態

**程序（動態）：**
- **定義**：程式的一個正在執行的實例
- **儲存**：以動態狀態存在於記憶體中
- **內容**：程式碼、資料、堆疊、堆積、暫存器、檔案描述符
- **狀態**：活動中，執行上下文不斷變化

```
程式檔案（在磁碟上）
       │
       ▼
   載入記憶體
       │
       ▼
   建立程序
       │
       ▼
   配置資源
       │
       ▼
   開始執行
```

#### **程序記憶體配置**

每個程序都有核心管理的明確定義的記憶體配置：

```
┌─────────────────────────────────────┐
│         堆疊（Stack）                │ ← 向下成長
│  （區域變數、函式呼叫）              │   - 函式呼叫框架
│                                    │   - 區域變數
│                                    │   - 返回位址
├─────────────────────────────────────┤
│         ↑                          │
│         │                          │
│         │                          │
│         │                          │
│         │                          │
├─────────────────────────────────────┤
│         堆積（Heap）                 │ ← 向上成長
│     （動態記憶體配置）                │   - malloc() 配置
│                                    │   - 動態資料結構
├─────────────────────────────────────┤
│        全域/靜態資料                  │ ← 固定大小
│      （全域變數等）                   │   - 全域變數
│                                    │   - 靜態變數
├─────────────────────────────────────┤
│           程式碼                     │ ← 唯讀
│        （程式指令）                   │   - 機器指令
│                                    │   - 常數
└─────────────────────────────────────┘
```

**記憶體管理原則：**

- **堆疊**：自動管理，隨函式呼叫而增長
- **堆積**：手動管理，隨動態配置而增長
- **資料**：固定大小，在程式啟動時初始化
- **程式碼**：唯讀，盡可能在程序之間共享

## 🔄 **程序建立與生命週期**

### **程序如何誕生**

Linux 中的程序建立涉及多個精密的步驟，將程式檔案轉換為正在執行的程序。這個過程稱為「分叉（forking）」，它建立一個父程序的副本，然後可以執行不同的程式碼或使用不同的資料執行相同的程式碼。

#### **Fork 哲學**

`fork()` 系統呼叫體現了**寫入時複製原則**——它創造了複製整個程序的假象，實際上在某個程序修改記憶體之前都是共享大部分記憶體的。這種最佳化對於嵌入式系統的效能至關重要。

**Fork 設計原則：**

- **效率**：在程序建立期間最小化記憶體複製
- **靈活性**：允許程序在建立後分歧
- **可靠性**：確保父程序和子程序之間的乾淨分離
- **效能**：針對立即 exec 的常見情況進行最佳化

#### **程序建立流程**

```
父程序
      │
      ▼
   fork() 系統呼叫
      │
      ▼
   核心建立程序描述符
      │
      ▼
   配置新的 PID
      │
      ▼
   複製父程序的記憶體描述符
      │
      ▼
   設定子程序專屬資料
      │
      ▼
   返回給兩個程序
      │
      ▼
   父程序：子程序 PID，子程序：0
```

**Fork 實作細節：**

1. **程序描述符**：核心配置一個新的 `task_struct`
2. **記憶體映射**：子程序獲得父程序記憶體描述符的副本
3. **檔案描述符**：子程序繼承已開啟的檔案和目錄
4. **訊號處理器**：子程序繼承訊號處理組態
5. **工作目錄**：子程序繼承目前的工作目錄

#### **基本程序建立範例**

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <sys/wait.h>

int main() {
    pid_t pid;
    
    printf("父程序啟動中 (PID: %d)\n", getpid());
    
    // 建立子程序
    pid = fork();
    
    if (pid < 0) {
        // Fork 失敗
        perror("Fork 失敗");
        exit(1);
    } else if (pid == 0) {
        // 子程序
        printf("子程序已建立 (PID: %d, 父程序 PID: %d)\n", 
               getpid(), getppid());
        
        // 子程序可以執行不同的程式碼
        printf("子程序執行中...\n");
        sleep(2);
        printf("子程序結束中\n");
        exit(0);
    } else {
        // 父程序
        printf("父程序繼續執行 (子程序 PID: %d)\n", pid);
        
        // 等待子程序完成
        int status;
        wait(&status);
        printf("子程序已完成，狀態碼: %d\n", WEXITSTATUS(status));
    }
    
    return 0;
}
```

**程序建立中的關鍵概念：**

- **返回值**：`fork()` 對父程序和子程序返回不同的值
- **記憶體共享**：父程序和子程序最初共享記憶體頁面
- **寫入時複製**：只有當某個程序進行寫入時，記憶體才會被複製
- **資源繼承**：子程序繼承大部分父程序的資源
- **PID 指派**：子程序獲得一個新的唯一程序 ID

## ⏱️ **程序排程**

### **在程序之間管理 CPU 時間**

程序排程是 Linux 核心決定在任何給定時間哪個程序應該在 CPU 上執行的機制。排程器必須平衡多個相互競爭的目標：公平性、回應性、吞吐量和資源利用率。

#### **排程哲學**

Linux 排程遵循**公平性原則**——所有程序都應該獲得合理的 CPU 時間份額，同時維持系統的回應性。排程器會適應不同類型的工作負載和系統需求。

**排程目標：**

- **公平性**：確保所有程序獲得合理的 CPU 時間
- **回應性**：最小化互動式程序的回應時間
- **吞吐量**：最大化整體系統效能
- **效率**：最小化排程開銷
- **可預測性**：為即時程序提供一致的行為

#### **Linux 排程器架構**

Linux 排程器在多個層級上運作：

```
┌─────────────────────────────────────┐
│         使用者程序                    │ ← 使用者空間
├─────────────────────────────────────┤
│         系統呼叫介面                  │ ← 邊界
├─────────────────────────────────────┤
│         排程器核心                    │ ← 核心空間
│         (CFS - 完全公平排程器)        │
├─────────────────────────────────────┤
│         CPU 排程器                   │ ← 硬體層級
│         （執行佇列管理）              │
└─────────────────────────────────────┘
```

**排程器組件：**

- **CFS（完全公平排程器）**：一般程序的主要排程器
- **即時排程器**：處理具有嚴格優先權的即時程序
- **負載平衡器**：在多個 CPU 之間分配程序
- **睡眠/喚醒邏輯**：管理等待事件的程序

#### **排程策略**

Linux 支援多種排程策略，每種都為特定使用情境而設計：

**SCHED_OTHER（一般排程）：**
- **用途**：大多數程序的預設策略
- **演算法**：完全公平排程器（CFS）
- **特性**：分時、動態優先權
- **使用情境**：一般應用程式、使用者程式

**SCHED_FIFO（即時先進先出）：**
- **用途**：執行到完成為止的即時程序
- **演算法**：基於優先權，無時間量子
- **特性**：可搶占、不主動讓出
- **使用情境**：硬即時應用程式

**SCHED_RR（即時循環制）：**
- **用途**：具有時間量子限制的即時程序
- **演算法**：基於優先權的時間分片
- **特性**：可搶占、相同優先權之間公平共享
- **使用情境**：軟即時應用程式

#### **排程策略範例**

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <sched.h>
#include <sys/resource.h>

int main() {
    int policy;
    struct sched_param param;
    
    // 取得目前的排程策略
    policy = sched_getscheduler(0);
    printf("目前的排程策略: ");
    
    switch (policy) {
        case SCHED_OTHER:
            printf("SCHED_OTHER（一般）\n");
            break;
        case SCHED_FIFO:
            printf("SCHED_FIFO（即時）\n");
            break;
        case SCHED_RR:
            printf("SCHED_RR（循環制即時）\n");
            break;
        default:
            printf("未知\n");
    }
    
    // 取得目前的優先權
    if (sched_getparam(0, &param) == 0) {
        printf("目前優先權: %d\n", param.sched_priority);
    }
    
    // 設定即時排程策略（需要 root 權限）
    param.sched_priority = 50;
    if (sched_setscheduler(0, SCHED_FIFO, &param) == 0) {
        printf("已成功設定為 SCHED_FIFO，優先權 50\n");
    } else {
        printf("無法設定即時排程（可能需要 root 權限）\n");
    }
    
    return 0;
}
```

**排程優先權管理：**

- **Nice 值**：範圍從 -20（最高優先權）到 +19（最低優先權）
- **即時優先權**：範圍從 1（最低）到 99（最高）
- **動態調整**：排程器根據行為調整優先權
- **優先權繼承**：防止優先權反轉問題

## 📡 **行程間通訊**

### **共享資料與協調執行**

行程間通訊（IPC）機制允許程序交換資料、同步它們的執行，並協調對共享資源的存取。Linux 提供了多種 IPC 機制，每種都針對特定的使用情境和效能需求而設計。

#### **IPC 設計哲學**

IPC 機制遵循**抽象化原則**——提供簡單、一致的介面，隱藏行程間通訊的複雜性。機制的選擇取決於應用程式的具體需求。

**IPC 選擇標準：**

- **效能**：資料需要多快速地傳輸？
- **可靠性**：資料完整性有多重要？
- **複雜度**：通訊需要多精密？
- **持續性**：通訊通道需要存在多久？
- **安全性**：程序之間需要多大程度的隔離？

#### **IPC 機制概覽**

```
┌─────────────────────────────────────┐
│         使用者應用程式                │
├─────────────────────────────────────┤
│         IPC 機制                    │
│  ┌─────────┬─────────┬─────────┐   │
│  │  管道   │  共享   │ 訊息    │   │
│  │         │  記憶體 │ 佇列    │   │
│  └─────────┴─────────┴─────────┘   │
├─────────────────────────────────────┤
│         核心支援                     │
│  （系統呼叫、記憶體管理）             │
└─────────────────────────────────────┘
```

**IPC 機制類型：**

- **管道（Pipes）**：簡單的單向位元組串流
- **FIFO**：用於非相關程序的具名管道
- **共享記憶體**：高效能資料共享
- **訊息佇列**：結構化訊息傳遞
- **Socket**：基於網路的通訊
- **訊號（Signals）**：非同步事件通知

#### **管道：簡單的資料流**

管道提供最簡單的 IPC 形式，在相關程序之間建立單向通訊通道：

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <string.h>

int main() {
    int pipefd[2];
    pid_t pid;
    char buffer[256];
    
    // 建立管道
    if (pipe(pipefd) == -1) {
        perror("管道建立失敗");
        exit(1);
    }
    
    pid = fork();
    
    if (pid < 0) {
        perror("Fork 失敗");
        exit(1);
    } else if (pid == 0) {
        // 子程序 - 寫入管道
        close(pipefd[0]); // 關閉讀取端
        
        const char *message = "來自子程序的問候！";
        write(pipefd[1], message, strlen(message) + 1);
        close(pipefd[1]);
        
        printf("子程序已發送訊息\n");
        exit(0);
    } else {
        // 父程序 - 從管道讀取
        close(pipefd[1]); // 關閉寫入端
        
        int bytes_read = read(pipefd[0], buffer, sizeof(buffer));
        if (bytes_read > 0) {
            printf("父程序收到: %s\n", buffer);
        }
        
        close(pipefd[0]);
        wait(NULL);
    }
    
    return 0;
}
```

**管道特性：**

- **單向性**：資料只能單方向流動
- **相關程序**：只有父子程序或相關程序可以使用
- **自動同步**：核心處理阻塞和緩衝
- **位元組串流**：沒有訊息邊界，只是連續的資料
- **核心緩衝**：資料在核心記憶體中進行緩衝

#### **共享記憶體：高效能資料共享**

共享記憶體透過允許多個程序存取同一區域的實體記憶體，提供最快的 IPC 機制：

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <sys/ipc.h>
#include <sys/shm.h>
#include <sys/wait.h>
#include <string.h>

int main() {
    key_t key = ftok("/tmp", 'A');
    int shmid;
    char *shared_memory;
    
    // 建立共享記憶體區段
    shmid = shmget(key, 1024, IPC_CREAT | 0666);
    if (shmid == -1) {
        perror("共享記憶體建立失敗");
        exit(1);
    }
    
    // 將共享記憶體附加到程序位址空間
    shared_memory = shmat(shmid, NULL, 0);
    if (shared_memory == (char *)-1) {
        perror("共享記憶體附加失敗");
        exit(1);
    }
    
    pid_t pid = fork();
    
    if (pid < 0) {
        perror("Fork 失敗");
        exit(1);
    } else if (pid == 0) {
        // 子程序 - 寫入共享記憶體
        strcpy(shared_memory, "透過共享記憶體來自子程序的問候！");
        printf("子程序已寫入共享記憶體\n");
        
        // 分離共享記憶體
        shmdt(shared_memory);
        exit(0);
    } else {
        // 父程序 - 從共享記憶體讀取
        wait(NULL);
        
        printf("父程序從共享記憶體讀取: %s\n", shared_memory);
        
        // 分離共享記憶體
        shmdt(shared_memory);
        
        // 移除共享記憶體區段
        shmctl(shmid, IPC_RMID, NULL);
    }
    
    return 0;
}
```

**共享記憶體特性：**

- **最高效能**：無需資料複製，直接存取記憶體
- **無核心開銷**：建立後不需要系統呼叫
- **需要同步**：程序必須協調存取
- **記憶體映射**：在程序位址空間中呈現為一般記憶體
- **系統範圍**：任何具有適當權限的程序都可以存取

## 🔒 **程序同步**

### **協調對共享資源的存取**

程序同步機制確保多個程序能夠安全地協調它們的執行並存取共享資源。Linux 提供了多種可跨程序邊界使用的同步原語。

#### **同步哲學**

同步遵循**安全性原則**——確保共享資源被安全地存取，同時維持系統效能並避免死鎖。

**同步目標：**

- **安全性**：防止競爭條件和資料損毀
- **活躍性**：確保程序能夠持續推進
- **效能**：最小化同步開銷
- **簡潔性**：使用滿足需求的最簡單機制
- **可靠性**：優雅地處理故障和邊界情況

#### **同步機制**

Linux 提供了多種基於 IPC 的同步機制：

**號誌（Semaphores）：**
- **用途**：資源計數和互斥
- **特性**：原子操作、可睡眠
- **使用情境**：資源池、生產者-消費者模式
- **實作**：System V 號誌、POSIX 號誌

**檔案鎖（File Locks）：**
- **用途**：基於檔案的互斥
- **特性**：建議性或強制性、程序範圍
- **使用情境**：檔案存取協調、資料庫鎖定
- **實作**：`flock()`、`fcntl()` 搭配 F_SETLK

**條件變數（Condition Variables）：**
- **用途**：等待特定條件
- **特性**：可睡眠、高效等待
- **使用情境**：生產者-消費者、屏障同步
- **實作**：POSIX 條件變數

#### **號誌實作範例**

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <sys/ipc.h>
#include <sys/sem.h>
#include <sys/wait.h>

int main() {
    key_t key = ftok("/tmp", 'C');
    int semid;
    
    // 建立一個包含一個號誌的號誌集
    semid = semget(key, 1, IPC_CREAT | 0666);
    if (semid == -1) {
        perror("號誌建立失敗");
        exit(1);
    }
    
    // 將號誌初始化為 1（用於互斥的二元號誌）
    union semun {
        int val;
        struct semid_ds *buf;
        unsigned short *array;
    } argument;
    
    argument.val = 1;
    if (semctl(semid, 0, SETVAL, argument) == -1) {
        perror("號誌初始化失敗");
        exit(1);
    }
    
    pid_t pid = fork();
    
    if (pid < 0) {
        perror("Fork 失敗");
        exit(1);
    } else if (pid == 0) {
        // 子程序
        struct sembuf operation = {0, -1, 0}; // 等待（遞減）
        
        printf("子程序等待號誌中...\n");
        if (semop(semid, &operation, 1) == -1) {
            perror("子程序: 號誌等待失敗");
            exit(1);
        }
        
        printf("子程序已取得號誌\n");
        sleep(2);
        
        operation.sem_op = 1; // 發送訊號（遞增）
        if (semop(semid, &operation, 1) == -1) {
            perror("子程序: 號誌發送訊號失敗");
            exit(1);
        }
        
        printf("子程序已釋放號誌\n");
        exit(0);
    } else {
        // 父程序
        struct sembuf operation = {0, -1, 0}; // 等待（遞減）
        
        printf("父程序等待號誌中...\n");
        if (semop(semid, &operation, 1) == -1) {
            perror("父程序: 號誌等待失敗");
            exit(1);
        }
        
        printf("父程序已取得號誌\n");
        sleep(1);
        
        operation.sem_op = 1; // 發送訊號（遞增）
        if (semop(semid, &operation, 1) == -1) {
            perror("父程序: 號誌發送訊號失敗");
            exit(1);
        }
        
        printf("父程序已釋放號誌\n");
        wait(NULL);
        
        // 移除號誌集
        semctl(semid, 0, IPC_RMID);
    }
    
    return 0;
}
```

**號誌操作：**

- **等待（P）**：遞減號誌，若為零則阻塞
- **發送訊號（V）**：遞增號誌，喚醒等待中的程序
- **原子性**：操作是不可分割的
- **阻塞性**：程序可以等待號誌的可用性
- **計數性**：可以表示多個可用資源

## 🔄 **程序狀態與轉換**

### **理解程序的生命週期**

Linux 中的程序可以存在於多種狀態中，每種狀態代表其生命週期的不同階段。理解這些狀態對於有效的程序管理和除錯至關重要。

#### **程序狀態哲學**

程序狀態代表**資源可用性模型**——程序根據其資源需求和系統可用性在各狀態之間移動。核心管理這些轉換以最佳化系統效能。

**狀態管理目標：**

- **效率**：最小化 CPU 閒置時間
- **公平性**：確保所有程序都能獲得 CPU 時間
- **回應性**：最小化互動式程序的回應時間
- **資源利用率**：有效利用可用資源
- **可預測性**：為即時程序提供一致的行為

#### **程序狀態圖**

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   已建立    │───▶│  可執行     │───▶│   執行中    │
│   (新建)    │    │ (就緒)      │    │ (活動中)    │
└─────────────┘    └─────────────┘    └─────────────┘
                           ▲                │
                           │                ▼
                    ┌─────────────┐    ┌─────────────┐
                    │   睡眠中    │◀───│   阻塞中    │
                    │  (等待中)   │    │ (I/O 等)    │
                    └─────────────┘    └─────────────┘
                           │                │
                           ▼                ▼
                    ┌─────────────┐    ┌─────────────┐
                    │   已停止    │    │   殭屍      │
                    │  (暫停中)   │    │ (已退出)    │
                    └─────────────┘    └─────────────┘
```

**程序狀態說明：**

- **已建立**：程序正在初始化
- **可執行**：程序已準備好執行，等待 CPU
- **執行中**：程序目前正在 CPU 上執行
- **睡眠中**：程序正在等待事件（I/O、訊號等）
- **已停止**：程序已被訊號暫停
- **殭屍**：程序已完成但退出狀態尚未被收集

#### **狀態轉換範例**

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <signal.h>
#include <sys/wait.h>

void signal_handler(int sig) {
    if (sig == SIGUSR1) {
        printf("程序 %d 收到 SIGUSR1\n", getpid());
    }
}

int main() {
    signal(SIGUSR1, signal_handler);
    
    printf("父程序 (PID: %d) 啟動中\n", getpid());
    
    pid_t pid = fork();
    
    if (pid < 0) {
        perror("Fork 失敗");
        exit(1);
    } else if (pid == 0) {
        // 子程序
        printf("子程序 (PID: %d) 已建立\n", getpid());
        
        // 子程序等待訊號（睡眠狀態）
        printf("子程序等待訊號中...\n");
        pause();
        
        printf("子程序在收到訊號後繼續執行\n");
        exit(0);
    } else {
        // 父程序
        printf("父程序繼續執行 (子程序 PID: %d)\n", pid);
        
        sleep(1);
        
        // 向子程序發送訊號（將其從睡眠狀態喚醒）
        printf("父程序向子程序發送 SIGUSR1\n");
        kill(pid, SIGUSR1);
        
        // 等待子程序完成
        int status;
        wait(&status);
        printf("子程序已完成，狀態碼: %d\n", WEXITSTATUS(status));
    }
    
    return 0;
}
```

**狀態轉換觸發條件：**

- **Fork**：已建立 → 可執行
- **排程器**：可執行 ↔ 執行中
- **I/O 請求**：執行中 → 睡眠中
- **I/O 完成**：睡眠中 → 可執行
- **訊號（SIGSTOP）**：執行中 → 已停止
- **訊號（SIGCONT）**：已停止 → 可執行
- **退出**：執行中 → 殭屍
- **等待**：殭屍 → 已終止

## 🚀 **進階程序管理**

### **超越基本程序操作**

進階程序管理涉及用於監控、控制和最佳化程序行為的精密技術。這些技術對於建構穩健、高效能的嵌入式系統至關重要。

#### **程序監控哲學**

程序監控遵循**可觀察性原則**——使系統行為可見且可理解，以便能夠快速識別和解決問題。

**監控目標：**

- **可見性**：了解程序正在做什麼
- **效能**：識別瓶頸和最佳化機會
- **除錯**：快速定位和解決問題
- **容量規劃**：了解資源需求
- **安全性**：偵測未經授權或可疑的活動

#### **程序資訊收集**

Linux 提供了多種收集程序資訊的機制：

**`/proc` 檔案系統：**
- **用途**：提供程序資訊的虛擬檔案系統
- **存取方式**：讀取 `/proc/<pid>/` 目錄中的檔案
- **資訊**：記憶體使用量、檔案描述符、環境變數等
- **即時性**：存取時的資訊是最新的

**系統呼叫：**
- **`getpid()`**：取得目前的程序 ID
- **`getppid()`**：取得父程序 ID
- **`getuid()`**：取得使用者 ID
- **`getgid()`**：取得群組 ID

**函式庫函式：**
- **`ps` 命令**：程序狀態資訊
- **`top` 命令**：即時程序監控
- **`strace` 命令**：系統呼叫追蹤

#### **程序控制範例**

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <signal.h>
#include <sys/wait.h>
#include <sys/types.h>
#include <sys/resource.h>

void print_process_info(const char *label) {
    printf("\n=== %s ===\n", label);
    printf("PID: %d\n", getpid());
    printf("父程序 PID: %d\n", getppid());
    printf("使用者 ID: %d\n", getuid());
    printf("群組 ID: %d\n", getgid());
    
    // 取得程序優先權
    int priority = getpriority(PRIO_PROCESS, 0);
    printf("優先權: %d\n", priority);
    
    // 取得資源使用量
    struct rusage usage;
    if (getrusage(RUSAGE_SELF, &usage) == 0) {
        printf("使用者 CPU 時間: %ld.%06ld 秒\n", 
               usage.ru_utime.tv_sec, usage.ru_utime.tv_usec);
        printf("系統 CPU 時間: %ld.%06ld 秒\n", 
               usage.ru_stime.tv_sec, usage.ru_stime.tv_usec);
        printf("頁面錯誤: %ld\n", usage.ru_majflt);
    }
}

void signal_handler(int sig) {
    printf("程序 %d 收到訊號 %d\n", getpid(), sig);
    
    if (sig == SIGUSR1) {
        printf("繼續執行...\n");
    } else if (sig == SIGTERM) {
        printf("正在優雅地終止...\n");
        exit(0);
    }
}

int main() {
    // 設定訊號處理器
    signal(SIGUSR1, signal_handler);
    signal(SIGTERM, signal_handler);
    
    print_process_info("父程序");
    
    pid_t pid = fork();
    
    if (pid < 0) {
        perror("Fork 失敗");
        exit(1);
    } else if (pid == 0) {
        // 子程序
        print_process_info("子程序");
        
        // 設定不同的優先權
        setpriority(PRIO_PROCESS, 0, 10);
        printf("子程序已將優先權設定為 10\n");
        
        // 等待訊號
        while (1) {
            printf("子程序等待訊號中...\n");
            sleep(5);
        }
    } else {
        // 父程序
        printf("父程序繼續執行 (子程序 PID: %d)\n", pid);
        
        sleep(2);
        
        // 向子程序發送訊號
        printf("父程序向子程序發送 SIGUSR1\n");
        kill(pid, SIGUSR1);
        
        sleep(2);
        
        printf("父程序向子程序發送 SIGTERM\n");
        kill(pid, SIGTERM);
        
        // 等待子程序終止
        int status;
        wait(&status);
        printf("子程序已終止，狀態碼: %d\n", WEXITSTATUS(status));
    }
    
    return 0;
}
```

**進階程序控制功能：**

- **優先權管理**：調整程序排程優先權
- **資源限制**：設定記憶體、CPU、檔案描述符的限制
- **訊號處理**：自訂對系統事件的回應
- **程序群組**：將程序組織成邏輯群組
- **工作階段管理**：控制終端機和工作控制

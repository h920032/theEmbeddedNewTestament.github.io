## 開機載入器的運作原理
### BIOS 為我們做了什麼
BIOS 是 PC 中 ROM 裡的韌體。當 PC 開機時，BIOS 是第一個執行的程式。所有其他程式都必須先載入到 RAM 中。BIOS 包含以下部分：

  * POST（開機自我測試）。計算主記憶體千位元組數的計數器是 POST 中最明顯可見的部分。
  * 設定選單，讓你設定一些參數並調整即時時鐘。大多數現代 BIOS 版本讓你設定開機順序，即 BIOS 檢查開機的裝置。這些可以是 A（第一個軟碟機）、C（第一個硬碟）、CD-ROM 以及可能的其他磁碟。列表中的第一個裝置將首先被嘗試。較舊的 BIOS 只有一個開機順序：A、C。所以 BIOS 會先嘗試從 A 開機，如果磁碟機中沒有磁片，則嘗試從 C 開機。
  * 開機磁區載入器。這會將開機磁碟的第一個 512 位元組磁區載入到 RAM 並跳轉到該位置。這就是本文所描述的開機載入器開始的地方。
  * BIOS 中斷。這些是簡單的裝置驅動程式，程式可以用來存取螢幕、鍵盤和磁碟。開機載入器依賴它們，大多數作業系統不使用（Linux 核心一旦啟動就不使用 BIOS 中斷）。MSDOS 確實使用 BIOS 中斷。
  
除了主 BIOS 之外，還有擴充 ROM，由主 BIOS 啟動。每個 VGA 顯示卡都有一個。SCSI 主機介面卡和乙太網路卡也可以有擴充 ROM。甚至可以在乙太網路卡上放置 EPROM，以便在沒有任何磁碟的情況下透過網路開機。

就開機載入功能而言，PC BIOS 與其他電腦系統相比是非常原始的。它對磁碟唯一了解的就是如何載入第一個 512 位元組磁區。

  軟碟的第一個磁區可以載入到位址 0000:7C00。磁區的最後兩個位元組會被檢查是否為 0x55 和 0xAA，這是一個粗略的完整性檢查。如果這些值正確，BIOS 就會跳轉到位址 0000:7C00。
  從硬碟開機與從軟碟開機非常相似。硬碟的第一個磁區（通常稱為主開機紀錄 MBR）被載入到 0000:7C00，然後 BIOS 跳轉到該位置。MBR 程式必須將自己移動到與 0000:7C00 不同的位址，因為它應該從分割區載入不同的開機磁區到位址 0000:7C00 並跳轉到該位置。
  現代 BIOS 版本可以將 CD-ROM 上的某個檔案視為軟碟映像。它們假裝從軟碟開機，將檔案的前 512 位元組載入到 0000:7C00 並跳轉到該位置。每次使用 BIOS 常式存取同一軟碟的嘗試，都會被重新導向到 CD-ROM 上的映像檔案。也可能支援其他從 CD-ROM 開機的方式（使用模擬硬碟或完全不使用磁碟模擬）。
當開機磁區被載入時，CPU 處於真實模式。對於不熟悉 80x86 架構的人來說：真實模式與 32 位元保護模式（Linux 執行的模式）相比非常受限。例如：64K 區段外的資料只有在改變區段暫存器時才能存取，而位址空間第一個 1MB 以外的資料（其中包含 640kB 的主記憶體）完全無法存取。由於 gcc 不了解真實模式，用它編譯的程式只能透過一些技巧和嚴格的記憶體大小限制在真實模式下執行。這就是為什麼大多數開機載入器（除了 GRUB）都是用組合語言撰寫的原因。所有開機磁區程式，即使是 GRUB 的，都是用組合語言撰寫的1。

理論上，開機載入器可以透過直接存取底層硬體來完成工作，但 512 位元組對此來說是非常小的空間。開機載入器可以存取 BIOS 中斷，這些是可以透過 INT 指令（軟體中斷）呼叫的子常式。這些只在真實模式下才能運作。大多數開機載入器使用以下常式。

  * INT 0x10 用於螢幕輸出。
  * INT 0x16 用於鍵盤輸入。
  * INT 0x13 用於磁碟 I/O。用於指定磁碟上磁區的參數過去範圍非常有限。最初只能在硬碟上指定 1024 個磁柱，而硬碟可能有更多磁柱。這對開機載入器和它存取的任何檔案可以放置的位置施加了限制。你被迫在硬碟開始處附近建立一個小分割區，並將開機載入器放在那裡。BIOS 有三個類別：
1995 年之前的 BIOS 版本只能存取約 500MB 的 IDE 磁碟，因為 BIOS 的磁區和磁頭編號直接對應到 IDE 介面上的暫存器值。
1995 年到 1998 年之間的 BIOS 版本可以存取最大約 8GB 的 IDE 磁碟。它們將 INT 0x13 呼叫中的磁柱、磁頭和磁區編號轉換為不同的值，以更好地利用 IDE 介面上的允許範圍。
1998 年或之後的 BIOS 版本有使用線性區塊位址的新呼叫介面。
在這三個類別中的任何一個，都有存在錯誤的 BIOS 導致它們在較低的磁碟大小限制處停止（例如一個應該支援最大 8GB 的第 2 類 BIOS，卻在 2GB 處停止）。在這些情況下，升級到新的 BIOS 版本是合理的。
  INT 0x15 是許多 BIOS 功能的通用中斷，其中之一是將資料移動到延伸記憶體（BIOS 需要暫時切換到保護模式來完成此操作）。其他功能包括停用 A20 閘道和確定記憶體大小。
### 開機載入器的組成部分
開機載入器通常由三個程式組成：

  * 開機磁區程式在開機時由 BIOS 直接載入，大小僅為 512 位元組。
  * 第二階段程式由開機磁區程式載入，它執行你期望開機載入器完成的所有工作。
  * 開機載入器安裝程式不在系統開機時執行，但它用於將開機載入器和第二階段程式安裝到開機磁碟上。這些必須儲存在特殊位置，因此不能用 cp 複製。
#### 開機磁區程式
開機磁區程式大小只能是 512 位元組，而且在所有情況下並非全部 512 位元組都可用。最後兩個位元組必須是 0x55 和 0xAA 以供 BIOS 使用。硬碟上的主開機紀錄包含 64 位元組的分割區表，因此只有前 446 位元組可以使用。如果開機磁區程式必須存在於 DOS 分割區或 DOS 軟碟上，則開機磁區的開頭必須有一個參數區塊。

由於這些大小限制，開機磁區程式幾乎是 PC 平台上僅存的真正針對大小最佳化的程式範例，並且因此必須用組合語言撰寫。此外，開機磁區程式無法完成你希望開機載入器做的所有事情。通常開機磁區程式會執行以下其中一項（不是在一個程式中同時執行三項）：

  * 載入另一個開機磁區。這對於存在於硬碟主開機紀錄中的開機磁區程式來說是典型的。它可以找到所選活動分割區的第一個磁區並鏈式載入它。傳統上隨 MS-DOS 附帶的 MBR 程式沒有在開機時更改活動分割區的能力。還有其他開機磁區程式讓你透過按鍵來選擇分割區，例如 LILO 的 MBR 程式。
  * 載入第二階段開機載入器。開機磁區程式通常無法在目錄中尋找特定名稱的檔案並將其載入記憶體，但也有例外，至少對於 DOS 檔案系統是如此2。大多數開機磁區程式透過磁區編號而非名稱來找到第二階段。磁區編號必須由開機載入器安裝程式寫入開機磁區。
  * 直接載入核心。核心通常比第二階段開機載入器大得多。

Linux 核心中的開機磁區程式直接將核心載入記憶體，無需第二階段開機載入器。由於核心位於軟碟上的連續磁區中，因此不需要遍歷檔案系統資料結構。但是，對於 bZimage 核心，開機磁區程式會作弊；它呼叫核心設定部分中的子常式，將核心的其餘部分載入到高記憶體。

開機載入器 e2boot 適合放入 ext2 分割區的第一個 1kB 區塊（它是開機磁區程式的兩倍大），但透過一些技巧，它可以在 ext2 分割區上按名稱找到核心和 RAM 磁碟並將它們載入記憶體。

此外，DOS 磁碟上的開機磁區也不使用第二階段開機載入器來載入 MS-DOS 核心檔案 IO.SYS 和 MSDOS.SYS。MSDOS 檔案系統的結構足夠簡單，可以在根目錄中找到特定名稱的檔案並將其載入記憶體，至少是部分載入。

#### 開機載入器的第二階段
這是真正的開機程式。它包含使用者介面和核心載入器。大小可以從 6.5 千位元組（LILO）到超過 100 千位元組（GRUB）不等。它包含以下功能：

  * 使用者介面。它可以是簡單的命令列（舊版本的 LILO）、選單或兩者兼備。它允許你選擇任意數量的作業系統並為作業系統指定額外的參數。可用選項由組態檔指定。現代版本的開機載入器可以在點陣圖圖片中顯示選單。
  * 作業系統載入器。將作業系統載入記憶體並執行它。或者我們可以載入另一個特定於其他作業系統的開機載入器並讓它執行。這稱為鏈式載入。

LOADLIN 不是完整的開機載入器，但它只有第二階段（沒有使用者介面）。它從 DOS 執行，可以利用 DOS 系統呼叫從磁碟讀取檔案。使它的任務比普通開機載入器更困難的是，它必須能夠從某些類型的記憶體管理器中脫身。

#### 開機載入器安裝程式
開機載入器的第三部分只在開機載入器安裝到磁碟時執行。與開機磁區程式和第二階段不同，這是一個普通的 Linux 程式。在 LILO 的情況下，每次組態更改或任何檔案更新時都必須重新執行安裝程式。它執行以下任務：

  * 安裝開機磁區。如果開機磁區將被安裝在硬碟的 MBR 中或 DOS 檔案系統上，並非所有 512 位元組都可以被覆寫，分割區表或 DOS 參數區塊必須被保留。
  * 告訴開機磁區第二階段開機載入器在哪裡。通常它會將一個或多個磁區位址寫入開機載入器。
  * 告訴第二階段開機載入器所有相關資訊在哪裡（組態、核心）。LILO 就是這種情況。LILO 建立一個映射檔，其中包含所有相關的磁區位址，並將指向映射檔的指標放入開機磁區和/或第二階段開機載入器中。

載入作業系統
### 載入 Linux 核心
#### Linux 核心由以下部分組成：

  * 開機磁區（參見 arch/i386/boot/bootsect.S）。這僅在 Linux 直接從軟碟開機時才執行。
  * 設定部分，包含真實模式初始化程式碼（轉換到保護模式）（參見 arch/i386/boot/setup.S）
  * 核心的其餘部分，依次由以下部分組成：
    * 保護模式初始化程式碼
    （參見 arch/i386/boot/compressed/head.S）。
    * 解壓縮程式（編譯過的 C 程式碼，參見
    arch/i386/boot/compressed/misc.c 和 lib/inflate.c）。

真正的核心是壓縮的。解壓縮後，這個核心由以下部分組成：
  * 保護模式初始化程式碼（參見 arch/i386/boot/head.S）。
  * 主 C 常式（參見 init/main.c）。
  * 所有其餘部分。與本討論相關的是 RAM 磁碟驅動程式（參見 drivers/block/rd.c）。這將在第 4 節中進一步說明。
  
Linux 開機載入器應該支援以下任務：

  * 將 Linux 核心載入記憶體。
    * 開機磁區（不會被執行）和核心的設定部分被載入到低記憶體的頂部附近（通常是位址 9000:0000）。
    * 如果不是 bzImage 類型的核心，核心的其餘部分將被載入到低記憶體的 0000:1000。
    * 如果是 bzImage 類型的核心，核心的其餘部分將被載入到高記憶體，從 0x100000 開始。

開機載入器不知道也不關心核心的其餘部分是否包含壓縮部分。
  將命令列傳遞給核心。此命令列可以來自組態檔，也可以由使用者互動輸入。
  將初始 RAM 磁碟載入記憶體並傳遞給核心。初始 RAM 磁碟將被載入到核心上方的高記憶體頂部附近。
  啟動核心。開機載入器必須在設定部分中設定一些相關參數並跳轉到該位置。從那裡核心取得控制權，開機載入器的任務就結束了。接下來核心按以下方式啟動：
    設定程式碼儲存相關參數和命令列以供後續使用。
    設定程式碼檢查可用的記憶體量，執行一些硬體初始化並準備進入保護模式。
    在保護模式程式碼中，核心的其餘部分被解壓縮3。
    解壓縮後，head.S 和主 C 常式將初始化核心的所有部分。這會顯示大量訊息。
    最後，init 程序將被啟動（或首先在初始 RAM 磁碟上啟動其 linuxrc 變體）。
#### 鏈式載入
大多數開機載入器設計為只開機一個作業系統。LILO 只知道如何載入 Linux 核心，而 DOS 開機磁區只能載入 DOS。如果你想在幾個不同的作業系統之間選擇，你不太可能找到一個可以載入所有作業系統的開機載入器。但至少每個作業系統都應該能被一個 512 位元組的開機磁區啟動，而這個開機磁區可以由 BIOS 載入，這就是關鍵。任何足夠先進的開機載入器都支援鏈式載入。

如果一個開機載入器載入另一個作業系統的開機磁區，這稱為鏈式載入。這個開機磁區可以直接從磁碟或分割區載入，也可以從檔案載入。對於另一個作業系統來說，其開機磁區是由 BIOS 還是由另一個開機載入器載入，應該沒有區別。實際上，分割區的開機磁區通常是由主開機紀錄程式載入的。在這種情況下，如果它是由更先進的開機載入器（如 LILO）而非其他載入，也應該沒有區別。

以下鏈式載入情境是可能的。

  Linux 開機載入器可以鏈式載入幾乎任何其他作業系統。
  Linux 開機載入器可以被其他作業系統的開機管理員鏈式載入。可以從 Windows NT 開機管理員或 OS/2 開機管理員啟動 Linux。
  Linux 開機載入器也可以鏈式載入其他 Linux 開機載入器。這在安裝了多個獨立 Linux 系統的電腦上可能有意義，其中每個 Linux 安裝都有自己的本地 LILO，而這些可以被一個中央 LILO 實例選擇來鏈式載入。LILO 實例甚至可以在它們之間交換命令列。
#### 設定開機載入器
LILO 和 GRUB 都有一個組態檔，其中指定了幾個選單選項，每個選項代表一個 Linux 核心或一個不同的作業系統來開機。對於每個 Linux 核心，可以指定命令列和初始 RAM 磁碟。除了語法細節外，這些組態檔的內容看起來非常相似。但有一個本質上的區別：

  LILO 在安裝時讀取其組態檔。每次組態檔、核心或任何初始 RAM 磁碟被更改時，都必須重新執行 LILO 安裝程式。在開機時不讀取組態檔。LILO 的第二階段開機程式不知道如何在檔案系統中找到檔案。它依賴映射檔來找到必要檔案的資料區塊。這個映射檔是由 LILO 安裝程式建立的。
  GRUB 以及 SYSLINUX 在開機時讀取它們的組態檔。你可以安裝一次開機載入器，然後毫無問題地更改組態檔、核心和 RAM 磁碟映像。第二階段開機程式知道如何在檔案系統中找到檔案。

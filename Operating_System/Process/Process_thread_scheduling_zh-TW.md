### 什麼是 PCB（程序控制區塊）？
與每個程序相關聯的資訊（也稱為任務控制區塊）。程序表的項目、Task Struct 或 Switchframe）是作業系統核心中的一種資料結構，包含管理特定程序排程所需的資訊。

* 程序狀態 – 執行中、等待中等
* 程式計數器 – 下一個要執行的指令位置
* CPU 暫存器 – 所有程序相關暫存器的內容
* CPU 排程資訊 – 優先權、排程佇列指標
* 記憶體管理資訊 – 分配給程序的記憶體
* 記帳資訊 – 已使用的 CPU、自啟動以來經過的時脈時間、時間限制
* I/O 狀態資訊 – 分配給程序的 I/O 裝置、開啟檔案清單

### 什麼是 TCB（執行緒控制區塊）？

與代表程序的程序控制區塊（PCB）非常相似，執行緒控制區塊（TCB）代表系統中產生的執行緒。它包含關於執行緒的資訊，例如其 ID 和狀態。

各組成元素定義如下：

* 執行緒 ID：這是作業系統在建立執行緒時分配給它的唯一識別碼。
執行緒狀態：這些是執行緒在系統中推進時所改變的狀態
* CPU 資訊：它包含作業系統需要知道的所有內容，例如執行緒已推進到什麼程度以及正在使用什麼資料。
* 執行緒優先權：它表示執行緒相對於其他執行緒的權重（或優先權），幫助執行緒排程器決定下一個應從就緒佇列中選取哪個執行緒。
* 指向觸發建立此執行緒之程序的指標。
* 指向由此執行緒建立之執行緒的指標。

### 排程準則

* CPU 使用率。我們希望讓 CPU 盡可能保持忙碌。從概念上講，CPU 使用率的範圍可以從 0 到 100%。在實際系統中，它應該在 40%（輕負載系統）到 90%（重度使用系統）之間。
* 吞吐量。如果 CPU 忙於執行程序，那麼工作正在完成。一個衡量工作量的指標是每單位時間完成的程序數量，稱為吞吐量。
* 周轉時間。從特定程序的角度來看，重要的準則是執行該程序需要多長時間。從程序提交到完成的時間間隔就是周轉時間。周轉時間是等待進入記憶體、在就緒佇列中等待、在 CPU 上執行以及進行 I/O 所花費時間的總和。
* 等待時間。CPU 排程演算法不影響程序執行或進行 I/O 的時間量，它只影響程序在就緒佇列中等待的時間量。等待時間是在就緒佇列中等待所花費時間的總和。
* 回應時間。從提交請求到產生第一個回應的時間。這個測量指標稱為回應時間，是開始回應所需的時間，而不是輸出回應所需的時間。

### 多執行緒的優點
多執行緒程式設計的優點可以分為四大類別：

* 回應性。對互動式應用程式進行多執行緒處理，即使程式的一部分被阻塞或正在執行冗長的操作，程式仍可繼續執行，從而提高對使用者的回應性。
* 資源共享。程序只能透過共享記憶體或訊息傳遞等技術來共享資源。此類技術必須由程式設計師明確安排。然而，執行緒預設會共享其所屬程序的記憶體和資源。
* 經濟性。為程序建立分配記憶體和資源的成本很高。由於執行緒共享其所屬程序的資源，因此建立和進行上下文切換執行緒更為經濟。雖然很難從經驗上衡量開銷的差異，但一般而言，建立和管理程序比執行緒更加耗時。
* 可擴展性。在多處理器架構中，多執行緒的優點可以大幅提升，因為執行緒可以在不同的處理器上平行執行。單執行緒程序只能在一個處理器上執行，無論有多少可用的處理器。在多 CPU 機器上進行多執行緒處理可以增加平行性。
